

# Проблема: отсутствие связанности между разделами отчёта

## Корневая причина

Текущая архитектура генерирует каждую категорию **параллельно и независимо** (строка 446, `Promise.all`). Каждый AI-вызов видит только биомаркеры своей категории. Поэтому:

- Раздел «Эндокринная система» не знает, что витамины B12/D уже сданы и есть в разделе «Обмен веществ»
- «Общее резюме» получает только по 2000 символов из каждого раздела (строка 750) — недостаточно для полной картины

## Предлагаемое решение: двухпроходная генерация

### Подход

1. **Передавать полный список всех сданных биомаркеров во все категории** — добавить в каждый категорийный промпт краткую сводку биомаркеров из других категорий (код + значение + статус), чтобы AI не рекомендовал сдавать то, что уже есть
2. **Увеличить контекст для резюме** — передавать полные тексты категорий (или хотя бы 5000+ символов вместо 2000) в промпт резюме
3. **Добавить инструкцию в промпт** — явно запретить рекомендовать дополнительные анализы на биомаркеры, которые уже присутствуют в сданной панели

### Конкретные изменения

**Файл: `supabase/functions/analyze-biomarkers/index.ts`**

1. **Сформировать глобальную сводку всех биомаркеров** (перед параллельными вызовами):
```text
ПОЛНЫЙ СПИСОК СДАННЫХ БИОМАРКЕРОВ ПАЦИЕНТА:
- B12: 450 пг/мл (🟢 ОПТИМАЛЬНО)
- Витамин D: 28 нг/мл (🟡 ДОПУСТИМО)
- ТТГ: 5.2 мМЕ/л (🟠 РИСК)
... (все биомаркеры из всех категорий)
```

2. **Добавить эту сводку в каждый категорийный промпт** с инструкцией:
```
ВАЖНО: Не рекомендуй сдавать анализы на биомаркеры, которые уже есть в списке выше. 
Вместо этого ссылайся на их значения.
```

3. **Увеличить лимит текста для резюме** с 2000 до 5000 символов на категорию

4. **Добавить в системный промпт резюме** инструкцию о согласованности выводов с детальными разделами

### Это решит

- Рекомендации сдать витамины, которые уже сданы
- Противоречия между резюме и детальными разделами (AI резюме будет видеть больше контекста)
- Общую несогласованность между разделами

### Не потребуется

- Менять архитектуру (параллельная генерация остаётся)
- Менять БД или фронтенд
- Увеличивать количество AI-вызовов
