Проблема: карточка биомаркера сейчас может целиком уехать на следующую страницу, если её верхний блок не помещается. Из-за этого сверху/снизу страницы остаётся большое пустое поле, как на скрине. Это не проблема данных и не мобильная адаптация — это правила page-break для карточек внутри Paged.js.

План исправления:

1. Исправить правила разрыва карточек биомаркеров
   - В `src/lib/reportLab/theme.css` убрать жёсткое удержание связки, которое заставляет карточку переноситься целиком.
   - Оставить карточке `.rl-biomarker` возможность дробиться между страницами.
   - Для верхней части карточки сохранить аккуратность: название + статус + шкала должны не разваливаться визуально, но не должны создавать огромную пустоту на предыдущей странице.

2. Настроить минимальные правила для пустых хвостов
   - Убрать/ослабить `break-after: avoid-page` там, где он провоцирует перенос следующей карточки.
   - Проверить `break-inside: avoid-page` на `.rl-bio-head` и `.rl-bio-scale`: при необходимости заменить на более мягкую структуру, чтобы перенос разрешался после шкалы/перед текстом.
   - Цель: если карточка не помещается целиком, на странице должна оставаться верхняя часть карточки, а текст продолжаться на следующей странице, вместо пустого поля.

3. Не менять адаптивность и данные
   - Не трогать parser, buildFromDb, список биомаркеров и HTML-структуру данных.
   - Не вводить разные CSS для mobile/desktop.
   - Рендер отчёта остаётся единым как PDF, отличается только scale.

4. Проверка
   - Прогнать тот же отчёт на mobile и desktop.
   - Проверить, что количество страниц и биомаркеров совпадает.
   - Проверить страницы вокруг проблемного места: перед HDL/следующими карточками не должно быть большого пустого хвоста.
   - Проверить, что карточки биомаркеров не исчезают и не меняют порядок.
