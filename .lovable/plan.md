

# План: Синхронизация причины назначения между AI и отчётом

## Текущая ситуация

Промпт `prescriptions_user` просит AI создавать назначения с полями:
- `prescription` — что делать
- `effect` — ожидаемый эффект  
- `duration_months` — длительность

**Не хватает:** поле `reason` (причина) с указанием конкретных биомаркеров.

## Что нужно изменить

### 1. Миграция БД — добавить поле `reason`

```sql
ALTER TABLE prescriptions ADD COLUMN reason TEXT;
```

### 2. Обновить промпт `prescriptions_user`

Добавить в формат JSON поле `reason`:

```text
ФОРМАТ JSON:
{
  "prescriptions": [{
    "prescription": "Витамин D 2000 МЕ, 1 раз в день",
    "reason": "25-OH Витамин D: 18 нг/мл (норма 30-50) — дефицит",
    "effect": "Нормализация уровня...",
    "duration_months": 3
  }]
}

ФОРМАТ ПРИЧИНЫ (reason):
- Указывай конкретный биомаркер и его значение
- Показывай норму в скобках
- Кратко описывай отклонение
- Примеры:
  * "Ферритин: 12 мкг/л (норма 30-150) — дефицит железа"
  * "ТТГ: 5.2 мМЕ/л (норма 0.4-4.0) — субклинический гипотиреоз"
  * "ЛПНП: 4.8 ммоль/л (норма <3.0) — повышен риск атеросклероза"
```

### 3. Обновить Edge Function

**supabase/functions/analyze-biomarkers/index.ts** (строки 774, 805-809, 823-832):

```text
Строка 774 — добавить reason в JSON формат:
  "{\"prescriptions\": [{\"prescription\": \"...\", \"reason\": \"...\", \"effect\": \"...\", \"duration_months\": ...}]}"

Строки 805-809 — добавить reason в валидацию:

Строки 823-832 — добавить reason в INSERT:
```
