## Что меняется

1. Кнопка «Запустить для всех» в карточке серии заменяется на **«Добавить пациентов»** — открывает модальное окно с выбором клиентов.
2. На каждую серию добавляется **отдельная вкладка** с детальным списком всех пациентов и их статусом по серии.

## 1. Модальное окно «Добавить пациентов в серию»

Новый компонент `EnrollPatientsDialog`:

- Список всех пациентов с поиском (имя, email, телефон), фильтрами:
  - Статус по этой серии: ещё не добавлен / уже в серии / отписался / завершил
  - Подписка: есть / нет
  - Email подтверждён: да/нет
- Чекбокс в каждой строке + чекбокс «Выбрать всех (с учётом фильтра)».
- Внизу панель: «Выбрано: N» + кнопка **Добавить в серию**.
- Опция: «База от регистрации» (по умолчанию: с момента добавления `now()`).
- По клику — вызов `drip-admin` с `action: 'enroll_users'` и массивом `user_ids`. Пациенты, уже имеющие записи в `email_drip_schedule` для этой серии, пропускаются (как сейчас в `enroll_all_existing`).
- Тост с количеством реально добавленных / пропущенных.

## 2. Вкладка «Подписчики» на уровне серии

В карточке каждой серии — внутренние табы: **Шаги** | **Подписчики (N)** | **Статистика**.

Подписчики — таблица с колонками:

| Пациент | Email | Статус в серии | Прогресс | Последнее письмо | Доставка | Отписка | Действия |

- **Статус в серии**: активен (есть pending) / завершено (все sent) / отменено / отписался / частично провалено.
- **Прогресс**: `sent / total` + мини-progress bar (например, 3/5).
- **Последнее письмо**: тема + дата отправки/планирования.
- **Доставка**: бейдж по `email_send_log.status` последнего письма (sent, delivered, bounced, complained, failed) — через джойн `email_send_log.metadata->>drip_schedule_id` или по `message_id`.
- **Отписка**: бейдж если email в `email_unsubscribes` (scope marketing/all).
- **Действия**: «Отменить оставшиеся», «Сбросить и перезапустить», «Открыть профиль».

Источники данных:
- `email_drip_schedule` агрегируется по `user_id, series_id` (counts по статусам, max `send_at`, последний `step_id`).
- `profiles` для имени/email.
- `email_unsubscribes` (lookup по email).
- `email_send_log` — джойн последнего лога по `metadata->>drip_schedule_id` (нужно добавить это поле при инсёрте — см. ниже).

Сверху вкладки: сводные плитки (Активны / Завершено / Отписались / Ошибки) + поиск + фильтр по статусу.

## 3. Бэкенд

В `supabase/functions/drip-admin/index.ts`:

- Новое действие `enroll_users` — принимает `series_id` и `user_ids: string[]`, делает batch insert через `enroll_user_in_series` с пропуском уже добавленных. Возвращает `{ enrolled, skipped }`.
- Оставить `enroll_all_existing` для обратной совместимости (вызывается изнутри, если пришёл флаг `all: true`).
- Действие `series_subscribers` — возвращает агрегированный список для вкладки «Подписчики» одной серии (чтобы не тащить тысячи строк в браузер; пагинация по 50, поиск/фильтр серверный).

В `drip-process` (минимальная правка): при создании `email_send_log` уже добавляется `metadata`; убедиться, что в `metadata` пишется `drip_schedule_id` — это нужно для джойна доставки. Если нет — добавить.

## Технические детали

- Серверный поиск/фильтр через `drip-admin` (action `series_subscribers`) с параметрами `{ series_id, search, status_filter, page, page_size }`.
- Для агрегата: один SQL в эдж-функции через `admin.from('email_drip_schedule').select('user_id, status, send_at, sent_at, step_id').eq('series_id', id)` + группировка в памяти эдж-функции (на серию обычно ≤ нескольких тысяч строк).
- Бейджи доставки используют существующие семантические токены (`bg-primary`, `text-muted-foreground`, `destructive` и т.д.) — без хардкод-цветов.
