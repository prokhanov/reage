

# Просчёт сценариев для плана стандартизации биологического возраста

## Предлагаемые формулы

```text
Формула health_index (серверная):
  health_index = 100
  для каждого биомаркера:
    midpoint = (normal_min + normal_max) / 2
    range = normal_max - normal_min
    deviation = |value - midpoint| / range
    
    if deviation < 0.1:  penalty = 0
    elif deviation < 0.2: penalty = 2 × aging_weight
    elif deviation < 0.5: penalty = 5 × aging_weight
    elif deviation < 1.0: penalty = 10 × aging_weight
    else:                 penalty = 20 × aging_weight
    
    health_index -= penalty
  
  health_index = clamp(health_index, 0, 100)

Формула bio_age:
  base_bio_age = chrono_age + (70 - health_index) × 0.3
  AI корректирует ±3 года макс
```

## Веса (aging_weight) по промпту

| Группа | Примеры маркеров | Вес |
|--------|-----------------|-----|
| Гликирование | HbA1c, Glucose | 2.0 |
| Воспаление | CRP, СОЭ, IL-6 | 1.8 |
| Гормоны | Testosterone, DHEA, Cortisol | 1.7 |
| Липиды | Cholesterol, LDL, HDL, Triglycerides | 1.5 |
| Метаболизм | Creatinine, ALT, AST | 1.3 |
| Энергия | Hemoglobin, Ferritin, Iron | 1.2 |

## Сценарий 1: Здоровый мужчина 40 лет, все 71 маркер в оптимуме

```text
Входные данные:
  chrono_age = 40, gender = male
  Все маркеры в пределах 10% от midpoint (deviation < 0.1)

Расчёт:
  Каждый маркер: penalty = 0
  health_index = 100

  base_bio_age = 40 + (70 - 100) × 0.3 = 40 + (-30 × 0.3) = 40 - 9 = 31.0
  AI коррекция: ±0..3 → финальный bio_age ≈ 31-34

Оценка: ПРОБЛЕМА — 31 год для 40-летнего даже при идеальных маркерах
кажется слишком оптимистичным. Разница -9 лет при health_index=100
слишком агрессивная.
```

**Корректировка формулы:**
```text
  base_bio_age = chrono_age + (70 - health_index) × 0.2
  
  Пересчёт:
  base_bio_age = 40 + (-30 × 0.2) = 40 - 6 = 34.0
  AI коррекция → 31-37

  Всё ещё агрессивно. Попробуем:
  base_bio_age = chrono_age + (70 - health_index) × 0.15
  
  base_bio_age = 40 + (-30 × 0.15) = 40 - 4.5 = 35.5
  AI коррекция → 32.5-38.5 ✓ Реалистично для идеально здорового
```

## Сценарий 2: Мужчина 45 лет, преддиабет + воспаление

```text
Входные данные:
  chrono_age = 45
  HbA1c = 6.2% (норма 4-5.6, midpoint=4.8, range=1.6)
    deviation = |6.2-4.8|/1.6 = 0.875 → penalty = 10 × 2.0 = 20
  Glucose = 6.8 (норма 3.9-6.1, midpoint=5.0, range=2.2)
    deviation = |6.8-5.0|/2.2 = 0.82 → penalty = 10 × 2.0 = 20
  CRP = 8.0 (норма 0-5, midpoint=2.5, range=5)
    deviation = |8-2.5|/5 = 1.1 → penalty = 20 × 1.8 = 36
  СОЭ = 22 (норма 0-15, midpoint=7.5, range=15)
    deviation = |22-7.5|/15 = 0.97 → penalty = 10 × 1.8 = 18
  Cortisol = 680 (норма 133-537, midpoint=335, range=404)
    deviation = |680-335|/404 = 0.85 → penalty = 10 × 1.7 = 17
  Testosterone = 10 (норма 12-33, midpoint=22.5, range=21)
    deviation = |10-22.5|/21 = 0.60 → penalty = 10 × 1.7 = 17
  LDL = 4.5 (норма 0-3.37, midpoint=1.69, range=3.37)
    deviation = |4.5-1.69|/3.37 = 0.83 → penalty = 10 × 1.5 = 15
  
  Остальные ~64 маркера в норме: ≈ 10 с мелкими отклонениями
    ~10 маркеров × penalty=2-5 × вес 1.2-1.5 ≈ 30

Суммарные штрафы: 20+20+36+18+17+17+15+30 = 173
health_index = max(0, 100 - 173) = 0

ПРОБЛЕМА: health_index обрушается в 0 при 7 плохих маркерах!
```

**Корректировка — нужно масштабирование:**
```text
При 71 маркере максимально возможный штраф ≈ 71 × 20 × 2.0 = 2840.
Нужен нормализующий делитель.

Вариант A: penalty делить на количество маркеров
  total_penalty = 173
  normalized_penalty = 173 / 71 × 30 = 73.1  (коэфф. 30 = эмпирический)
  health_index = 100 - 73 = 27

  base_bio_age = 45 + (70-27) × 0.15 = 45 + 6.45 = 51.5
  AI коррекция → 48.5-54.5

Вариант B: средний penalty по маркерам × множитель
  avg_penalty = 173 / 71 = 2.44
  health_index = 100 - avg_penalty × 25 = 100 - 61 = 39

  base_bio_age = 45 + (70-39) × 0.15 = 45 + 4.65 = 49.7
  AI коррекция → 46.7-52.7

Оба варианта дают реалистичные 49-52 для больного 45-летнего ✓
```

## Сценарий 3: Женщина 55 лет, всего 5 маркеров (мало данных)

```text
Входные данные:
  chrono_age = 55
  Glucose = 4.8 (в норме) → penalty ≈ 0
  Hemoglobin = 135 (в норме) → penalty ≈ 0
  СОЭ = 10 (в норме) → penalty ≈ 0
  ALT = 22 (в норме) → penalty ≈ 0
  Cholesterol = 5.0 (в норме) → penalty ≈ 0

  total_penalty = 0 из 5 маркеров
  Но 66 маркеров ОТСУТСТВУЮТ!

Без коррекции: health_index = 100, bio_age = 55-8.25 = 46.75
ПРОБЛЕМА: ложный оптимизм при 7% данных
```

**Решение — confidence penalty:**
```text
  coverage = присутствующие маркеры / всего маркеров = 5/71 = 7%
  confidence_factor = min(1.0, coverage / 0.5) = 0.14

  Применяем: health_index стремится к 70 при low coverage
  adjusted_health_index = 70 + (raw_health_index - 70) × confidence_factor
  adjusted_health_index = 70 + (100 - 70) × 0.14 = 70 + 4.2 = 74.2

  base_bio_age = 55 + (70-74.2) × 0.15 = 55 - 0.63 = 54.4
  AI коррекция → 51.4-57.4 ✓ Реалистично — чуть лучше хронологического
```

## Итог: финальные параметры формулы

```text
Коэффициент bio_age:  0.15 (не 0.3 — слишком агрессивно)
Нормализация штрафов: avg_penalty × 25 (средний штраф по маркерам)
Confidence threshold:  50% покрытия = полная уверенность
Якорь при low data:   health_index → 70 (bio_age ≈ chrono_age)
AI коррекция:         ±3 года максимум
```

## Рекомендуемые изменения к плану

1. **Коэффициент 0.15** вместо 0.3 — иначе разброс слишком большой
2. **Нормализация штрафов** — делить суммарный penalty на количество маркеров, умножать на 25
3. **Confidence factor** — health_index стремится к 70 при малом покрытии
4. **Coverage threshold** — 50% маркеров = полная уверенность в расчёте
