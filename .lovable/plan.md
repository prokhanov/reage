## Проблема

В `src/contexts/DemoModeContext.tsx` после прошлого этапа было ошибочно разрешено демо на собственном служебном аккаунте админа/врача/суперадмина.

Правило: демо-данные бывают только в пациентском контексте. Админ/врач/суперадмин на своём аккаунте не видит и не включает демо. Админ в кабинете пациента может делать всё то же, что пациент, включая демо-режим.

## Что делаю

1. Добавить единую функцию принятия решения по доступу к demo mode:
   - свой аккаунт пациента → можно;
   - свой аккаунт staff (`admin`, `doctor`, `superadmin`) → нельзя, stale `demo_mode_enabled` сбрасывается;
   - view-as пациента → можно;
   - view-as не пациента → нельзя, чужой флаг не трогаем.
2. Применить это правило в `fetchDemoModeStatus` и `toggleDemoMode`.
3. Оставить проверку «есть реальные анализы» после проверки доступа.
4. Поправить комментарий баннера: он показывается только когда контекст уже разрешил demo mode.

Больше ничего не трогаю: UI профиля, схему БД, RLS, edge-функции.

## Регресс-проверка

- Админ/врач/суперадмин на своём аккаунте с `demo_mode_enabled=true` → демо не показывается, флаг сбрасывается в `false`, баннера нет.
- Админ/врач/суперадмин на своём аккаунте пытается включить демо → отказ с текстом про служебный аккаунт.
- Админ в режиме «просмотр как пациент» → тумблер работает для этого пациента; если у пациента уже есть анализы — по-прежнему блок с текстом про анализы.
- Пациент без анализов включает демо → работает как раньше.
- Пациент с анализами → по-прежнему блок «У вас уже есть реальные анализы».

## Логик-тест (подводные камни)

- Юнит-тест на helper: patient own allowed; admin/doctor/superadmin own forbidden; view-as patient allowed; view-as non-patient forbidden без очистки чужого флага.
- Grep по `demo_mode_enabled` и комментариям — не должно остаться правила «админ на своём аккаунте видит демо».
- `viewAsUserId` логика не меняется — тумблер пишет в профиль просматриваемого пациента.

## Что НЕ входит

- Не меняю UI, тексты, схему БД, RLS, edge-функции.
- Не меняю правило «нельзя включить, если уже есть реальные анализы» — это защита данных, а не роль.
