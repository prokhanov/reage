## Что выяснилось

- Тестовое письмо из админки ушло (`email_send_log`: `signup / sent` для `prokhanov@gmail.com` в 17:36) — инфраструктура очереди работает.
- Реальная регистрация `a.s.prokhanov@yandex.ru` в 17:22: в `email_send_log` **нет ни одной строки `signup`** для этого адреса — только drip/welcome.
- Логи edge-функции `send-verification-email` пусты — функция **ни разу не была вызвана**, хотя `Register.tsx` её зовёт.

## Причина

1. Функция `send-verification-email` создана в прошлом ходе, но **не задеплоена** (новые функции из чата требуют явного deploy). Запрос с фронта уходит в 404 → ловится `.catch`, тостов не показывает.
2. Вызов сделан как fire-and-forget (`.invoke().catch(...)`) **прямо перед `goToStep(2)`**. Даже после деплоя при быстрой смене стейта/ремаунте запрос может не успеть улететь — нет ожидания.

## План

1. **Задеплоить** `send-verification-email` и (на всякий случай) `confirm-email-token`.
2. В `src/pages/Register.tsx`:
   - Сделать вызов **await** (с таймаутом ~5с) — чтобы гарантированно отправить до перехода шага. Ошибку логируем, но регистрацию не блокируем (тост: «Письмо подтверждения отправим повторно из ЛК»).
3. Проверка: зарегистрировать нового пользователя → в `email_send_log` должна появиться строка `template_name='signup', status='sent'` для его email, в логах функции — запись «token created / enqueued».

Ничего не трогаем: шаблон, `auto_confirm_email`, `confirm-email-token`, drip, профили.
