

# Конкретизация назначений

## Проблемы

1. **Комплексные назначения вместо отдельных** — "Антиоксиданты (CoQ10 + Витамин С + Витамин Е)" вместо 3 отдельных строк
2. **Нет формы выпуска** — промпт не требует указывать форму (хелатные, бисглицинатные и т.д.), не запрещает дешёвые формы
3. **Продукты вместо добавок** — "ешьте шпинат" вместо "Витамин К2 (MK-7) 100 мкг"
4. **Нет превентивных назначений** — код отправляет AI только аномальные маркеры (`abnormalBiomarkers`), AI не видит нормальные показатели и не может назначить поддерживающие дозы
5. **Промпт слишком мягкий** — "только если нужны", "может быть 0 назначений"

## Что меняем

### A. Код edge function `analyze-biomarkers/index.ts`

Добавить новый плейсхолдер `{allBiomarkers}` — передавать AI **все** сданные маркеры (не только аномальные), чтобы AI мог назначить превентивные добавки для маркеров в норме, но на нижней границе оптимума.

### B. Промпт `prescriptions_system` (UPDATE в БД)

Расширить роль: добавить указание о превентивных протоколах и биодоступных формах.

### C. Промпт `prescriptions_user` (UPDATE в БД)

Ключевые изменения:
- **Одна добавка = одно назначение** (запрет комплексов в одной строке)
- **Обязательно указывать форму выпуска** (хелат, бисглицинат, метилфолат и т.д.)
- **Запрет дешёвых форм**: оксид магния → бисглицинат/таурат, цианокобаламин → метилкобаламин, фолиевая кислота → метилфолат
- **Питание = дополнение, не замена** добавки. Если маркер отклонён — назначай добавку + питание отдельными пунктами
- **Превентивные назначения**: даже если все в норме — магний, омега-3, витамин D, витамин К2 как базовый протокол, но с обоснованием ("поддержание оптимальных уровней", "на нижней границе оптимума")
- **Добавить `{allBiomarkers}`** — полная картина для обоснования превентивных назначений
- Конкретные примеры формата: "Магний бисглицинат 400 мг, 1 капсула перед сном, курс 3 месяца"

### Объём работ
- 1 изменение в edge function (добавить `allBiomarkers` + новый плейсхолдер)
- 2 UPDATE в `ai_prompt_settings` (system + user промпты)
