
# План: Упростить рендеринг Markdown — использовать стандартную библиотеку

## Суть проблемы
Текущий кастомный рендерер `ol` в `MarkdownContent.tsx` пытается вручную нумеровать списки и определять подзаголовки эвристиками. Это ломается на нестандартном AI-контенте.

## Решение
Отказаться от кастомной логики нумерации и использовать стандартное поведение `react-markdown`. Для обработки подзаголовков — чистить markdown **до** рендеринга в `cleanMarkdownArtifacts`.

---

## Технические изменения

### 1. Обновить препроцессор `src/lib/markdown.ts`

Добавить логику преобразования подзаголовков **до** парсинга markdown:

```typescript
// Найти паттерн:
// "11. ...текст...\n\n12. Подзаголовок\n\n13. Текст после"
// Где "12. Подзаголовок" — короткая фраза без точки в конце (2-7 слов)
// Преобразовать в:
// "11. ...текст...\n\n### Подзаголовок\n\nТекст после" (абзац без номера)

// Регулярка для определения пунктов-подзаголовков:
preprocessed = preprocessed.replace(
  /^(\d+)\.\s+([А-ЯЁA-Z][^\n.!?:;,]{3,50})$/gm,
  (match, num, text) => {
    const words = text.trim().split(/\s+/);
    // Подзаголовок: 2-7 слов, без пунктуации в конце
    if (words.length >= 2 && words.length <= 7 && !/[.!?:;,]$/.test(text.trim())) {
      return `### ${text.trim()}`;
    }
    return match;
  }
);

// Убираем номера у пунктов сразу после подзаголовка (делаем абзацем)
preprocessed = preprocessed.replace(
  /(### [^\n]+\n\n)(\d+)\.\s+/g,
  '$1'
);
```

### 2. Упростить `src/components/MarkdownContent.tsx`

Убрать кастомный `ol` рендерер, вернуть стандартный:

```typescript
ol: ({ children }) => (
  <ol className="list-decimal list-outside pl-6 mb-4 space-y-2 text-foreground">
    {children}
  </ol>
),
li: ({ children }) => (
  <li className="text-foreground leading-relaxed">
    {children}
  </li>
),
```

Удалить функции `getTextContent` и `isLikelySubheading` — они больше не нужны.

### 3. Обновить PDF-экспорт `src/pages/Recommendations.tsx`

Аналогичная логика в `parseMarkdownToPdfMake`:
- Детектировать подзаголовки как пункты с коротким текстом без пунктуации
- Рендерить их как заголовки (`h3` style)
- Следующий пункт после подзаголовка — как абзац без номера

---

## Результат

- Стандартная нумерация списков (1., 2., 3...) без кастомного кода
- Подзаголовки типа "Системные взаимосвязи" автоматически становятся заголовками `###`
- Текст после подзаголовка — обычный абзац (не пункт списка)
- Меньше кода = меньше багов
