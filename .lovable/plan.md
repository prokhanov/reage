Цель: при отправке формы «Напишите нам — Ответим в течение часа в рабочее время» отправлять цель Яндекс.Метрики `form1` (счётчик 109706546). Лендинг v2 (`/landing-v2`) не трогаем.

## Наблюдение
Форма «Напишите нам» существует ровно **одна** — `src/components/landing/FeedbackDialog.tsx`. Её открывают все триггеры на боевом лендинге `/`:
- кнопка «Оставить заявку» в hero (`HeroPortrait.tsx`, в т.ч. её мобильные/повторные копии),
- ссылка «напишите нам» в `FAQSection.tsx`,
- кнопка в `CTASection.tsx`.

Поэтому достаточно одной точки интеграции — на успешной отправке формы. Это отработает и на десктопе, и на мобиле, для любого способа открытия.

## Изменения

1. **`src/lib/yandexMetrika.ts`** — новая утилита:
   ```ts
   export const YM_COUNTER_ID = 109706546;
   export function reachGoal(goal: string, params?: Record<string, unknown>) {
     if (typeof window !== "undefined" && typeof window.ym === "function") {
       window.ym(YM_COUNTER_ID, "reachGoal", goal, params);
     }
   }
   ```
   Тип `window.ym` уже есть в `src/vite-env.d.ts`.

2. **`src/components/landing/FeedbackDialog.tsx`** — в `handleSubmit` после успешного ответа `supabase.functions.invoke("send-feedback", …)` (там, где сейчас показывается тост «Сообщение отправлено») вызвать `reachGoal("form1")`. Единственная точка — вся форма «Напишите нам».

## Что не меняем
- Кнопки на страницах `/landing-v2` — по вашему указанию.
- Другие формы (`ConsultationCtaBlock` «бесплатная консультация», `ExampleReportDialog` и т.п.) — это не форма «Напишите нам»; если понадобится — сделаем отдельной задачей.

## Проверка
- Сабмит формы «Напишите нам» на проде должен фиксироваться в Метрике как достижение цели `form1` (Отчёты → Цели). До событий может быть задержка до нескольких минут.
- Клиентская проверка: в консоли `window.ym` — функция, при сабмите видно вызов; блокировщики Метрики могут её глушить — это ожидаемо.
