## Цель

Зафиксировать правильное поведение demo mode:

- У админа/врача/суперадмина на собственном аккаунте demo mode выключен и не используется.
- В режиме просмотра пациента админ видит именно demo mode этого пациента.
- В режиме просмотра пациента админ может включать и выключать demo mode пациента.
- Баннер demo mode показывается в режиме просмотра пациента, если у пациента включён demo mode.

## Причина текущей ошибки

`PatientViewDialog` создаёт `ViewAsPatientProvider`, но внутри него нет отдельного `DemoModeProvider`. Из-за этого страницы в модалке просмотра пациента используют внешний demo-context админа. Поэтому переключение внутри кабинета пациента может попадать в логику собственного служебного аккаунта и показывать тост «Демо-режим недоступен».

Текущая функция `resolveDemoModeAccess` сама по себе уже содержит правильное правило: в `view-as` пациент разрешён, собственный staff-аккаунт запрещён. Но этот контекст не применяется внутри модалки пациента.

## Что изменить

1. В `src/components/admin/PatientViewDialog.tsx`:
   - импортировать `DemoModeProvider`;
   - обернуть содержимое внутри `ViewAsPatientProvider` в отдельный `DemoModeProvider`;
   - добавить `DemoBanner` в верхнюю область модалки просмотра пациента, чтобы при включённом demo mode пациента баннер был виден и там.

2. Создать небольшой внутренний компонент для layout модалки пациента, который сможет вызвать `useDemoMode()` уже внутри нового `DemoModeProvider`:
   - `demoMode === true` → показать `DemoBanner`;
   - кнопка отключения баннера вызывает `toggleDemoMode(false)` и выключает demo mode именно у просматриваемого пациента.

3. Не менять бизнес-правило:
   - staff на собственном аккаунте не может включить demo mode;
   - patient на собственном аккаунте может;
   - admin/doctor/superadmin в `view-as` пациента может.

## Регресс после этапа

- Собственный аккаунт админа: demo mode не включается, demo-баннера нет.
- Просмотр пациента `team@reage.life`: если у пациента demo mode включён — видны demo-данные и баннер.
- В профиле пациента внутри view-as тумблер demo mode включает/выключает demo mode пациента.
- Отключение demo mode из баннера внутри view-as выключает demo mode пациента, а не админа.
- Обычный пациент без view-as сохраняет прежнее поведение.
