# Профиль — мобильная доработка

Только UI/верстка `src/pages/Profile.tsx` и `src/components/profile/PhoneChangeField.tsx`. Логика, запросы и диалоги не трогаем.

## Что не так сейчас (мобилка 390px)

1. Заголовки карточек обрезаются в 2–3 строки, потому что кнопка «Редактировать» (с иконкой + текст) рядом с круглой 48px иконкой не помещается. Видно у «Личная информация», «Паспортные данные», «История болезней».
2. Каждое поле (Имя/Email/Пол/Дата/Рост) — отдельная карточка с рамкой и `p-4`. На узком экране это лишний шум и большая длина прокрутки.
3. Строка телефона: `+7 (...` обрезается, бейдж «Не подтверждён» переносится в 2 строки, кнопка «Изменить» жмётся к краю.
4. Контейнер `py-8 space-y-6 p-6` на каждой карточке — на мобиле слишком воздушно, контент уезжает вниз.
5. Кнопка «Выйти» в destructive-красном — выглядит тревожно для рутинного действия.
6. Заголовок страницы `text-3xl` дублирует крупные заголовки карточек, иерархия плоская.

## План изменений

### 1. Контейнер страницы
- `px-4 py-8` → `px-4 pt-4 pb-8 sm:py-8`
- `space-y-6` → `space-y-4 sm:space-y-6`
- Заголовок «Профиль»: `text-3xl` → `text-2xl sm:text-3xl`, `mb-8` → `mb-5 sm:mb-8`.

### 2. Шапка каждой карточки (Личная / Паспорт / История / Демо / Безопасность)
Единый паттерн — иконка меньше, заголовок не обрезается, кнопка-иконка на мобиле:

```text
[icon 40px]  Заголовок              [✎]
             Подзаголовок
```

- Круг иконки: `w-12 h-12` → `w-10 h-10 sm:w-12 sm:h-12`, иконка `h-6 w-6` → `h-5 w-5 sm:h-6 sm:w-6`.
- Заголовок: `text-xl` → `text-lg sm:text-xl`, добавить `leading-tight`.
- Кнопка «Редактировать»: на мобиле — `size="icon"` (только карандаш, `aria-label="Редактировать"`), на `sm:` возвращается с текстом. Скрывать/показывать через `sm:hidden` / `hidden sm:inline-flex`.
- Карточка: `p-6` → `p-4 sm:p-6`.

### 3. Личная информация — поля списком
Заменить сетку из 5 бордерных мини-карточек на компактный список с разделителями:

```text
Имя              Алина Дарбинян
Email            alina@mail.ru
Пол и возраст    Женщина · 37 лет
Дата рождения    23 октября 1988
Рост             164 см
```

- Контейнер: `divide-y divide-border/40 rounded-lg border border-border/40 bg-background/40`.
- Строка: `flex items-center justify-between gap-3 px-4 py-3 text-sm`, лейбл `text-muted-foreground`, значение `font-medium text-right truncate`.
- На `sm:` оставить текущую 2-колоночную сетку с бордер-карточками (как сейчас).
- Иконки рядом с лейблом — убрать на мобиле (избыточно), на `sm:` оставить.

### 4. Блок телефона (`PhoneChangeField`)
- Display-state: вертикальный stack на мобиле — строка `+7 …` на всю ширину без обрезки, под ней ряд `[бейдж статуса] [Изменить]`.
- Использовать `flex flex-col gap-2 sm:flex-row sm:items-center sm:justify-between`.
- Бейдж «Не подтверждён»: `whitespace-nowrap`.
- Кнопка «Изменить»: на мобиле `variant="ghost" size="sm"` с иконкой `Edit2`, выровнена по правому краю ряда.

### 5. Паспортные данные
- Пустое состояние: `p-4` → `p-3`, текст уже компактный — оставить как есть; убедиться, что кнопка «Заполнить» на всю ширину на мобиле (`w-full sm:w-auto`).
- Заполненное состояние: серия/номер — те же 2 поля; на мобиле сделать одну строку через тот же divide-list паттерн (Серия / Номер), на `sm:` — текущая 2-колоночная сетка.

### 6. Демо-режим
- Текст внутри Switch-карточки: `p-4` → `p-3 sm:p-4`, описание `text-xs sm:text-sm`.
- Подпись «Демо-режим» внутри карточки убрать (дублирует заголовок секции) — оставить только описание возле свитча.

### 7. Безопасность / Выход
- Заменить `variant="destructive"` на `variant="outline"` с красным текстом и иконкой: `text-destructive border-destructive/40 hover:bg-destructive/10`. Менее агрессивно, как принято в качественных приложениях (Apple/Linear pattern).
- Подзаголовок «Управление аккаунтом» убрать на мобиле через `hidden sm:block` — он ничего не добавляет.

### 8. Карточка «Следующий анализ»
- `p-6` → `p-4 sm:p-6`, дата `text-lg` → `text-base sm:text-lg`.

## Технические детали

- Все правки — Tailwind классы, никаких новых зависимостей.
- Используем существующие semantic tokens (`text-muted-foreground`, `border-border/40`, `bg-background/40`, `text-destructive`).
- `EditProfileDialog`, `EditPassportDialog`, `EditMedicalHistoryDialog`, `MedicalAnketaCard` — не трогаем (отдельный скоуп).
- Проверка: повторный screenshot 390×1800 после правок, сравнение шапок карточек на отсутствие переноса.

## Чего НЕ делаем

- Не меняем структуру данных, запросы, диалоги, демо-режим, логику телефона.
- Не редизайним десктоп — поведение `sm:` и выше сохраняется как есть.
- Не трогаем `MedicalAnketaCard` (отдельный компонент, отдельная задача).
