План правки только для `HeroBlockPortrait.tsx`:

1. **Зафиксировать всю правую композицию как один артборд**
   - Заменить текущий `w-full + h` блок на внутренний фиксированный canvas для каждого брейкпойнта.
   - На мобиле/планшете/десктопе будут не плавные проценты и растягивание, а несколько заранее заданных размеров: например mobile, tablet, desktop.

2. **Запретить растягивание карточек**
   - Убрать responsive-width классы вида `w-[...] sm:w-[...] lg:w[...]` с самих карточек там, где они создают «живое» изменение размера.
   - Задать карточкам конкретные ширины внутри каждого canvas-сценария, чтобы при изменении экрана они не становились шире/уже между контрольными размерами.

3. **Зафиксировать мужика и карточки в одной системе координат**
   - Портрет и виджеты будут позиционироваться относительно фиксированного контейнера, а не относительно всей доступной ширины.
   - Сам контейнер будет центрироваться/масштабироваться только ступенчато, чтобы выглядел как статичная собранная картинка.

4. **Переделать mobile/tablet ближе к референсу**
   - На мобиле: вертикальная композиция как в приложенном примере — человек справа/внизу, карточки аккуратно поверх/рядом, без ухода в отдельные блоки и без касаний.
   - На планшете: отдельная проверенная раскладка, не промежуточная «резиновая» версия.

5. **Проверить пересечения**
   - После реализации проверить минимум 3 ширины: телефон, планшет, текущий desktop-preview.
   - Цель: ни одна карточка не касается другой, карточки не меняют размер плавно, композиция выглядит как единая статичная картинка.
