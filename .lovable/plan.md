## Что меняем (только `src/pages/AnalysisDetail.tsx`)

### Шапка анализа
- Стрелка «назад» в одну строку с заголовком, заголовок переносим на мобиле к `text-xl md:text-3xl`, убираем агрессивный градиент с длинной даты — оставляем `text-foreground font-semibold` (градиент мешает читать), бейдж статуса опускаем на новую строку под датой на мобиле.
- Подпись «Сдано маркеров: N · Лаб.» — `text-xs md:text-sm`.
- Кнопки админ-режима («Редактировать», «Перегенерировать») оставляем как есть (видны только в view-as / у админа на десктопе).

### Поиск
- Контейнер `px-3 sm:px-4`, инпут `h-11 text-base`, плейсхолдер укорачиваем: «Поиск маркера…».

### Аккордеон категории
- Полностью копируем визуал `AccordionItem` / `AccordionTrigger` из `src/pages/Biomarkers.tsx` (строки 321–363): `rounded-2xl`, иконка категории через `getBiomarkerCategoryIcon(category)`, название слева с бейджем количества, без перевода по центру. Бейдж «балл/score» из Биомаркеров не показываем — у AnalysisDetail нет per-analysis score-категорий, в худшем случае вытащим из `analysis.biomarkers_metadata.category_scores` если он есть; если нет — просто не рендерим правый блок.

### Тело категории
- На мобиле (`md:hidden`) — список карточек один-в-один как в Биомаркерах (строки 366–433):
  - название + код,
  - значение + единица справа, цвет = `getStatusHslColor(status)`,
  - `BiomarkerStatusBadge` снизу слева,
  - дата анализа справа (берём `analysis.date`, тренда тут нет — пропускаем `getTrendIcon`),
  - `BiomarkerScale compact` снизу, если есть `min`/`max`.
  - Описание показываем через `<Tooltip>` на иконке `Info` в правом верхнем углу карточки (или раскрывающимся текстом — реализуем через `Info` `Tooltip` как сейчас в таблице).
- На десктопе (`hidden md:block`) — оставляем существующую таблицу 5 колонок без изменений.

### Технические детали
- Никакой бизнес-логики не трогаем: `loadData`, `handleAnalyze`, прогресс-диалог, отчёт — без изменений.
- Импорты: добавить `getBiomarkerCategoryIcon` из `@/lib/categoryIcons` (уже используется в Биомаркерах).
- Колонка `latest_date` в Биомаркерах = дата анализа; здесь явно `analysis.date` для всех значений категории.
