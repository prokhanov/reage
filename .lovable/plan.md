## Цель

К моменту шага «Подписка» данные тарифов уже должны быть в кеше — без спиннера и без скачка верстки.

## Что делаем

### 1. Прелоад данных тарифов с самого начала регистрации
В `src/pages/Register.tsx` вызвать `useSubscriptionPlans()` на верхнем уровне страницы (один раз при монтировании, ещё на шаге 1). Запрос уйдёт в фоне параллельно с заполнением формы; React Query закеширует результат по ключу `['subscription-plans', false, false]`. Когда юзер дойдёт до шага оплаты, `RegisterStep5` вызовет тот же хук с тем же ключом и получит данные из кеша мгновенно — без повторного запроса и без `isLoading=true`.

Дополнительно: задать в этом хуке `staleTime: 5 * 60 * 1000` (5 минут), чтобы при переходах между шагами данные не считались устаревшими и не было фонового рефетча, дёргающего ререндер карточек.

### 2. Убрать «мигание» и разъезжание верстки в `RegisterStep5`
Сейчас при загрузке рендерятся `Skeleton h-[420px]`, а затем карточки — высоты разные, popover-кнопка внутри карточки меняет ширину, отсюда layout shift в узком превью.

Изменения в `src/components/register/RegisterStep5.tsx`:
- Скелетоны заменить на блоки с такой же структурой и `min-height`, как у реальных карточек (одинаковая высота во всех брейкпоинтах).
- На контейнер-сетку повесить `min-h-[...px]`, чтобы при переключении skeleton→cards высота не прыгала.
- Карточкам задать `min-h-0` + `h-full` и `min-w-0` на колонках сетки (`grid-cols-1 md:grid-cols-3` + `min-w-0` на внутреннем `div`), чтобы длинный текст/число биомаркеров не растягивал карточку по ширине в узком вьюпорте.
- Цены и подписи обернуть `truncate` / `whitespace-nowrap` корректно, без `flex-wrap`, ломающего колонку.

### 3. Не трогаем
- Логику выбора плана, оплату Robokassa, «Оплатить позже», «Сравнить тарифы», активную подписку.
- Серверную часть, edge functions, БД — изменений нет.

## Технические детали

- `useSubscriptionPlans` — без изменения сигнатуры, только `staleTime`.
- Прелоад — обычный вызов хука в `Register.tsx`. Никакого `queryClient.prefetchQuery` не нужно, т.к. React Query сам делит кеш по ключу.
- Никаких изменений в маршрутизации и шагах.

## Результат

- Шаг 5 открывается с уже готовыми карточками тарифов.
- Нет мигания skeleton → cards.
- Верстка не разъезжается в узком превью Lovable.
