Причина найдена.

Что произошло:
- В свежем тексте секции «Метаболизм и Детоксикация» сейчас 0 открывающих biomarker-якорей и 0 закрывающих `biomarker_end`.
- Вместо якорей модель сгенерировала обычные Markdown-заголовки вида `### Уробилиноген (UBG)`.
- Рендерер карточек не строит карточку по заголовку — он строит её только по `<!-- anchor:biomarker CODE -->`.
- Валидатор должен был восстановить якоря, но текущая функция восстановления пропускает Markdown-заголовки: она ищет plain-text строки биомаркера и специально исключает строки, начинающиеся с `###`.
- Это сломалось именно после того, как в промптах разрешили/закрепили `###`-заголовки для биомаркеров: формат стал допустимым для генерации, но валидатор не был адаптирован под этот формат.

Почему кажется, что валидатор проверяет меньше:
- Он сейчас ограничен по времени и по числу AI-догенераций, чтобы не зависать.
- Главное: если якорей нет вообще, часть проверок карточек просто не к чему применить. Валидатор видит голый текст с заголовками, но не превращает эти заголовки в карточки.

План исправления:
1. Обновить восстановление якорей в `report-qa`, чтобы оно распознавало `### Название (CODE)` / `## Название (CODE)` как начало карточки биомаркера.
2. Вставлять `<!-- anchor:biomarker CODE -->` перед таким заголовком и `<!-- anchor:biomarker_end -->` перед следующим биомаркером или перед блоком общей оценки.
3. Убрать конфликт в промптах: не писать, что `biomarker_end` должен быть один раз только в Блоке 4; закрытие должно быть у каждой карточки либо валидатор должен корректно ставить границы.
4. Добавить страховку в рендерер: если якорей нет, но есть заголовки `### ... (CODE)`, всё равно показывать карточки, а не голый текст.
5. Проверить на текущем отчёте Проханова/последней секции «Метаболизм и Детоксикация», что после запуска валидатора появляются карточки и список пропущенных маркеров реально считается.
