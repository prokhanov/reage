Сделаем это как временный инструмент для настройки 3-го блока на `/landing-v2`, без изменения реального поведения лендинга для обычных посетителей.

План:
1. Добавить в `HeroBlockPortrait` режим редактирования позиций карточек, включаемый только параметром URL, например:
   ` /landing-v2?layoutEdit=1 `
2. В этом режиме карточки внутри артборда можно будет перетаскивать мышью/пальцем отдельно для текущего брейкпойнта:
   - mobile
   - tablet
   - desktop
3. Сделать панель координат поверх страницы:
   - показывает текущий брейкпойнт
   - показывает `top / left / width / rotate` для каждой карточки
   - кнопка «Скопировать координаты» копирует готовый JSON/TS-объект
   - кнопка «Сбросить» возвращает текущие координаты из кода
4. Хранить изменения временно в `localStorage`, чтобы при обновлении страницы расстановка не пропадала.
5. В обычном режиме без `?layoutEdit=1` лендинг останется как сейчас: статичные карточки, без панели и без перетаскивания.
6. После того как вы пришлёте мне скопированные координаты для mobile/tablet/desktop, я вторым шагом захардкожу их в компонент и уберу временный edit-режим.

Технически:
- Меняем только `src/components/landing/v2/HeroBlockPortrait.tsx`.
- Карточки останутся абсолютно позиционированными внутри фиксированных артбордов.
- Для редактирования добавим небольшой внутренний `DraggableHeroWidget`/helper прямо в этом файле, без новых библиотек.
- Координаты будут считаться в пикселях артборда, чтобы потом их можно было напрямую перенести в `style={{ top, left, width, transform }}`.
