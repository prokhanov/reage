

## Анализ: Приоритет диапазонов (Общий vs Гендерные)

### Текущая логика (неявная)

В `biomarkerNorms.ts` при режиме `general`:
1. Если есть `normal_min_male` → используется мужской диапазон
2. Иначе → используется `normal_min` (общий)

То есть гендерные значения **всегда приоритетнее** общих. Общий — это fallback. Но в UI это никак не обозначено.

### Проблема

Если заполнены все три (Общий, Мужчины, Женщины), непонятно что реально используется. Админ может заполнить общий диапазон, думая что он применяется ко всем, а система его игнорирует для пациентов с указанным полом.

### Предлагаемое решение

Добавить визуальную индикацию в UI редактирования:

1. **Подсказка под секцией "Общие диапазоны"**: текст «Гендерные значения приоритетнее общих. Общий диапазон используется только если гендерный не задан.»

2. **Визуальная маркировка**: если для конкретного уровня (крит/норма/опт) заполнены и общий, и гендерный — общий показывать с пониженной opacity и пометкой «fallback», гендерные — как основные.

3. **Альтернатива**: добавить `gender_mode` поле (`'general_only' | 'gender_priority'`) по аналогии с `range_mode`, чтобы админ мог явно выбрать: использовать только общие или гендерные приоритетнее.

### Рекомендация

С медицинской точки зрения гендерные диапазоны **всегда** должны быть приоритетнее — это стандарт лабораторной диагностики. Делать это переключаемым — риск ошибки. Лучше просто **явно показать приоритет в UI** (подсказка + визуальное затенение fallback-значений).

### Изменения

**Файл: `src/pages/admin/DataManagement.tsx`**
- Добавить информационную подсказку над секцией диапазонов: «Приоритет: Гендерные → Общие (fallback)»
- Для каждого ряда (critical, normal, optimal): если гендерное значение заполнено, показывать общее поле с `opacity-50` и подписью «fallback»
- Никаких изменений в БД и логике оценки — приоритет остаётся тем же
