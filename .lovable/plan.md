

## Проблема

В БД в `demo_report_system` и `demo_report_user` лежат **старые** промпты с инструкциями про подпункты "Что это", "Зачем нужен", "Интерпретация результата". Хардкод-константы в коде обновлены, но `loadData()` сразу перезаписывает стейт значениями из БД — поэтому генерация всегда использует старые промпты из БД.

## План

### 1. Обновить промпты в БД через миграцию

Выполнить SQL UPDATE для `demo_report_system` и `demo_report_user` — записать туда актуальные тексты из хардкод-констант (нарративный стиль, без списков, без подзаголовков "Что это" / "Зачем нужен").

### 2. Очистить `demo_report_result`

Старый результат тоже сгенерирован по старому промпту — обнулить его (`prompt_text = ''`), чтобы при загрузке страницы не показывался устаревший отчёт.

Изменения только в БД, код не меняется — хардкод-константы уже актуальны и служат fallback.
