Цель: пока пользователь печатает, текст ниже сразу съезжает на следующие страницы — как в Google Docs. Никаких «применить на blur».

## Как сейчас
- `PagedReportPreview` один раз прогоняет `Previewer` (Paged.js) и вставляет готовые страницы в DOM.
- В режиме edit блоки `[data-editable-id]` становятся `contenteditable` прямо внутри уже свёрстанной страницы.
- При наборе Paged.js ничего не пересчитывает: текст растёт внутри своей «коробки», нижние страницы стоят на месте, лишние строки визуально уходят под колонтитул/за край и в PDF пропадают.
- Пересборка происходит только при blur через полный ререндер Previewer — это и есть источник жалобы.

## Что делаем
1. Реалтайм-репагинация при каждом изменении текста
   - На `input` в редактируемом блоке: собираем HTML всех фрагментов этого `data-editable-id` (Paged.js мог его разрезать), конвертируем в markdown, кладём в drafts.
   - Дебаунс ~120–200мс, чтобы при быстром наборе не гонять пересборку на каждый символ, но визуально это ощущалось как «живое».
   - drafts прокидываются в `PagedReportPreview` → `Previewer` перегоняет весь `ReportDocument` заново → страницы пересобираются, весь текст ниже автоматически съезжает.

2. Плавность как в Google Docs
   - Пересборку делать в фоне: готовим новый DOM в оффскрин-контейнере, потом одним swap подменяем `.pagedjs_pages`. Никаких «моргающих» страниц.
   - Сохраняем и восстанавливаем: позицию курсора (по offset внутри редактируемого блока), выделение, скролл вьюпорта.
   - После swap возвращаем фокус в тот же `data-editable-id` и ставим caret в сохранённое место (даже если блок теперь разрезан по-другому — считаем offset по plain-text).

3. Никакого blur-flow для содержимого
   - Убираем «финальное сохранение по blur» как способ увидеть новую разметку.
   - Blur оставляем только для скрытия floating toolbar.
   - Кнопка «Сохранить» продолжает писать в БД, но верстка на экране уже актуальна в любой момент.

4. Защита от гонок
   - Один активный `Previewer.preview()` за раз; если во время пересборки пришёл новый input — отменяем предыдущую (флагом cancelled) и стартуем новую с самым свежим drafts.
   - Пока идёт первая пересборка, редактор остаётся живым: пользователь продолжает печатать в текущем DOM, при swap мы просто применим последний drafts.

5. Проверка
   - `/admin/report-visuals` → «Редактировать».
   - В блоке ближе к концу страницы дописать 5–10 строк подряд.
   - Убедиться, что: текст ниже съезжает в реальном времени, курсор не прыгает, скролл не убегает, ничего не теряется на границе страниц, PDF-режим по-прежнему без экранных зазоров.

## Технические детали
- Файлы: `src/lib/reportLab/renderer/PagedReportPreview.tsx` (реалтайм-input handler, дебаунс, оффскрин swap, сохранение caret/scroll), `src/lib/reportLab/editor/ReportEditorContext.tsx` (drafts уже есть, менять API не нужно).
- Caret: сохраняем как `{editableId, textOffset}` через обход текстовых узлов; после swap находим блок с тем же id и восстанавливаем `Range` по offset.
- Debounce: `requestIdleCallback` + fallback `setTimeout(150)`.
- Оффскрин-рендер: временный `div` с `position:absolute; visibility:hidden`, `Previewer.preview()` в него, затем `output.replaceChildren(...newPages)`.
