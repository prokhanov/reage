

# Согласованность общего резюме с разделами и назначениями

## Текущие разрывы

1. **Резюме не видит `{globalBiomarkers}`** — работает только с обрезанными текстами категорий (5000 символов). Сырые значения и статусы маркеров недоступны.
2. **Резюме генерируется ДО назначений** — не может ссылаться на конкретные назначения.
3. **Назначения не видят резюме** — могут противоречить приоритетам, заявленным в резюме.
4. **5000 символов на категорию** — длинные отчеты (10000+ символов) обрезаются, теряя рекомендации из конца раздела.

## Что меняем

### A. Добавить `{globalBiomarkers}` в промпт резюме

В edge function: добавить `.replace(/{globalBiomarkers}/g, globalBiomarkersSummary)` при формировании `summaryPrompt` (строка 810-812).

В промпте `summary_user`: добавить блок с полной панелью маркеров, чтобы резюме могло ссылаться на конкретные значения.

### B. Добавить `{categoryRecommendations}` в промпт резюме

Извлечь рекомендательные секции (anchor:actions) из категорийных отчетов — ту же логику, что уже работает для назначений — и передать в резюме. Это гарантирует, что резюме видит все конкретные рекомендации из разделов.

### C. Увеличить лимит с 5000 до 8000 символов на категорию

Строка 800: `report.substring(0, 8000)` — покроет 95%+ отчетов без обрезки.

### D. Изменить порядок: сначала назначения, потом резюме

Переставить блоки генерации: назначения → резюме. Передать в резюме список назначений через новый плейсхолдер `{prescriptionsList}`. Резюме сможет ссылаться на конкретные назначения ("вам назначен магний бисглицинат 400 мг...").

В промпте `summary_user` добавить:
```
НАЗНАЧЕННЫЕ ДОБАВКИ И РЕКОМЕНДАЦИИ:
{prescriptionsList}

Упоминай конкретные назначения в контексте приоритетов.
```

### Объем работ
- Edge function: переставить блоки (назначения перед резюме), добавить 3 плейсхолдера в summaryPrompt, увеличить лимит
- 1 UPDATE в `ai_prompt_settings` (`summary_user`) — добавить блоки `{globalBiomarkers}`, `{categoryRecommendations}`, `{prescriptionsList}`
