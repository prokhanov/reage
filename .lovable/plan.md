## Проблема
В `src/pages/admin/UserManagement.tsx` удаление сотрудника/приглашения работает без подтверждения — клик по иконке корзины сразу вызывает `handleDeleteUser`. Отсюда «случайные» удаления и ощущение глюка. Плюс после успешного вызова `delete-user` используется локальный `refetch()` вместо инвалидации React Query кэша, из-за чего список иногда показывает удалённого пользователя до перезагрузки страницы, и приходится «удалять ещё раз».

## Что делаем
Только фронт `src/pages/admin/UserManagement.tsx`. Edge-функцию `delete-user` и БД не трогаем.

1. **AlertDialog подтверждения** (по образцу `Patients.tsx`):
   - Общий диалог, который хранит `pendingDelete: { id, name, type, inviteToken } | null` в состоянии.
   - Клик по иконке корзины больше не вызывает `handleDeleteUser` напрямую — только выставляет `pendingDelete` и открывает диалог.
   - Разные тексты для активного пользователя и pending-приглашения:
     - Активный: «Удалить пользователя {name}? Все его данные (роли, разрешения, история, анализы если есть) будут удалены безвозвратно.»
     - Приглашение: «Удалить приглашение для {name}? Ссылка перестанет работать.»
   - Кнопка «Удалить» в `bg-destructive`, кнопка «Отмена».

2. **Устойчивость удаления**:
   - Ввести `deletingId` state, чтобы блокировать кнопку и показывать «Удаление…» пока идёт запрос — исключает двойные клики.
   - После успеха: `queryClient.invalidateQueries({ queryKey: ["admin-users"] })` вместо `refetch()`, чтобы гарантированно перестроить список и сбросить кэш.
   - На ошибке `delete-user` выводить `data.error`/`error.message` в toast и оставлять пользователя в списке (сейчас логика уже такая, только уточним сообщение — например «Не удалось удалить: …»).

3. **Мелочи**:
   - `handleSuspendUser` тоже переключить с `window.confirm` на тот же AlertDialog-подход (одно поле `pendingSuspend`), чтобы стиль был консистентный. Тексты: «Приостановить доступ для {name}? Все административные роли будут сняты, пациентская роль сохранится.»
   - Убрать голый `console.log('User deleted successfully', …)` — оставить только на случай ошибки.

## Проверка
- Клик по корзине у активного сотрудника → диалог → «Отмена» ничего не делает; «Удалить» удаляет, кнопка блокируется, список обновляется без ручного refresh.
- Клик по корзине у pending-инвайта → диалог с текстом про приглашение → удаление сразу очищает строку.
- Двойной клик по «Удалить» в диалоге не запускает второй запрос (кнопка disabled).
