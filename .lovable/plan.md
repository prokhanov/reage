## Цель
Перенести кнопку «Пересчитать и проверить» (доступна только суперадмину в режиме просмотра пациента) со страницы «Стратегия здоровья» на «Мое здоровье» (`/dashboard`). Расширить диалог превью: помимо био-возраста и таймлайна показывать разбивку Индекса здоровья и рейтинги систем организма с пояснениями.

## Что меняется

### 1. Дашборд (`src/pages/Dashboard.tsx`)
- Добавить в шапку рядом с заголовком кнопку «Пересчитать и проверить» (видна только если `isSuperAdmin && isViewMode`, по аналогии с условием в HealthStrategy).
- Подключить `StrategyPreviewDialog`, состояния `previewOpen / previewData / generating / publishing`, обработчики `openPreview` и `publishEdited` (перенести из HealthStrategy).
- После успешной публикации перезагружать данные дашборда (bio age, HI, рейтинги систем).

### 2. Стратегия здоровья (`src/pages/HealthStrategy.tsx`)
- Удалить кнопку «Пересчитать и проверить», подключение диалога и обработчики `openPreview/publishEdited` — переезжают в Dashboard.
- Импорты и состояния, ставшие неиспользуемыми, убрать.

### 3. Edge Function `compute-health-strategy`
В режиме `preview: true` дополнить ответ:
- `explanation.health_index` — уже есть (value, top_deviations); добавить `breakdown`: среднее по маркерам, top-5 штраф, бонус за ≥80% оптимальных, итоговая формула `100 − (0.6·avg + 0.4·top5)·12`.
- `explanation.system_ratings` — массив `{ category, score, rationale }` для 5 систем (источник — тот же AI-вызов, что используется на дашборде; либо подтянуть последний сохранённый снапшот рейтингов).
Режим `publish: true` сохраняет отредактированные значения как сейчас.

### 4. Диалог превью (`src/components/health-strategy/StrategyPreviewDialog.tsx`)
Добавить новые секции (collapsible, с короткими подписями «почему так»):
- **Био-возраст** — формула, AI-дельта, драйверы (как сейчас).
- **Индекс здоровья** — значение, разбивка штрафов, топ-отклонения, бонус.
- **Рейтинги систем организма** — таблица 5 систем со score и пояснением; чекбокс «принять как есть» / поле редактирования score.
- **Таймлайн и ожидания** — как сейчас, с чекбоксами исключения.
Кнопки «Отменить» и «Сохранить и опубликовать клиенту» остаются.

## Технические детали
- Условие отображения: `isSuperAdmin && isViewMode` — берётся из контекста просмотра пациента (хук, который уже используется в HealthStrategy).
- `StrategyPreviewDialog` остаётся в текущей папке (`src/components/health-strategy/`) — переиспользуется с Dashboard.
- При публикации перерасчитанные рейтинги систем сохраняются в `health_strategy_snapshots` (новое поле `system_ratings jsonb`, если ещё не предусмотрено — потребуется миграция; иначе используем существующее поле снапшота).
- На стороне Dashboard после `publishEdited` дёргаем существующие лоадеры (bio age, HI, system ratings), чтобы UI пациента сразу отразил изменения.

## Что НЕ меняется
- Логика расчёта био-возраста и HI (якорь 85, slope 0.20, штрафы 25/8, AI ±5) — без изменений.
- Видимость для обычных пациентов: кнопка скрыта, ничего не меняется.
- Страница «Стратегия здоровья» продолжает показывать опубликованные данные.
