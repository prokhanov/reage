
# План: Исправление дублирования отчётов

## Текущая проблема

В таблице `recommendations` для анализа Тестова обнаружено **14 записей** вместо 7 — каждый тип отчёта продублирован:
- Первая генерация: `15:06:00`
- Вторая генерация: `15:21:23`

**Причина:** Edge function `analyze-biomarkers` не очищает старые recommendations перед генерацией новых.

## Решение

### Шаг 1: Очистка текущих дубликатов

Использую `cleanup-prescriptions` или создам запрос для удаления старых рекомендаций (по timestamp `15:06:00`):

```sql
-- Удалить записи первой генерации (оставить новые)
DELETE FROM recommendations 
WHERE analysis_id = '61a46d28-2792-4e24-afbe-97e5acf710df'
AND created_at < '2026-02-05 15:21:00+00'
```

### Шаг 2: Добавить защиту в analyze-biomarkers

В начале edge function добавлю удаление существующих recommendations перед генерацией:

```typescript
// Удалить старые рекомендации перед генерацией новых
await supabase
  .from("recommendations")
  .delete()
  .eq("analysis_id", analysisId);
```

Это предотвратит накопление дубликатов при повторных запусках AI-анализа.

## Технические детали

### Изменения в файлах:

**supabase/functions/analyze-biomarkers/index.ts** (~строки 85-90):

```text
Добавить после получения анализа:
├─ Проверка существования анализа
├─ DELETE FROM recommendations WHERE analysis_id = ?
└─ Продолжение генерации
```

## Результат

После выполнения:
- Дубликаты в текущем отчёте будут удалены (7 записей вместо 14)
- При повторной генерации старые записи автоматически удаляются
- Dropdown секций отчёта показывает каждый тип только один раз
