## Цель
В блоке Hero (`src/components/landing/v2/HeroBlockPortrait.tsx`) виджеты сейчас плавно «плывут» по `sm:`/`lg:`/`xl:` брейкпойнтам — из-за этого на промежуточных ширинах карточки наезжают друг на друга и на части тела на портрете. Нужно перейти на 3 фиксированные, заранее проверенные раскладки, и убрать «растягивание» карточек.

## Что меняем

1. **Три дискретных раскладки вместо плавных классов**
   - Mobile: `< 640px` (базовые классы)
   - Tablet: `sm:` (≥ 640px)
   - Desktop: `lg:` (≥ 1024px)
   - Убираем промежуточные `xl:` сдвиги координат и любые «полу-шаги» — оставляем ровно 3 набора `top/left/right/bottom/width`.

2. **Фиксированные размеры карточек (не тянутся)**
   - Каждый виджет получает фиксированную `width` (в px) для каждого из 3 брейкпойнтов. Никаких `flex-1` / `w-full` на самих карточках.
   - Внутренние шкалы/строки уже фиксированы — оставляем.

3. **Мобильная раскладка по референсу (скрин amoCRM)**
   - Все 4 виджета **видимы на мобилке** (сейчас Biomarkers и Systems скрыты через `hidden sm:block` — это и есть «вынос в отдельные блоки»). Убираем `hidden sm:block`.
   - Виджеты накладываются на портрет (как на референсе), но **не пересекаются между собой**. Раскладка по референсу:
     - BioAge — слева, на уровне груди
     - Biomarkers — правее и ниже BioAge, не касается его
     - Recommendations — ниже Biomarkers, ближе к правому краю
     - Systems — слева внизу, под BioAge
   - Между любыми двумя соседними карточками — зазор ≥ 12px по вертикали и горизонтали.

4. **Проверка непересечения через Playwright**
   - После правок снимаем скриншоты на 3 ширинах: 375, 768, 1280.
   - Парсим bounding box каждой карточки, проверяем что прямоугольники не пересекаются (с минимальным зазором 8px). Если пересекаются — корректируем координаты до тех пор, пока тест не пройдёт на всех трёх ширинах.

5. **Что НЕ трогаем**
   - Содержимое карточек (тексты, иконки, цвета, внутреннюю вёрстку) — без изменений.
   - Левую колонку (заголовок, кнопки, StatRow, бейдж локации) — без изменений.
   - Десктопную раскладку оставляем близкой к текущей, только нормализуем под единый `lg:` шаг без `xl:`.

## Технические детали

Файл: `src/components/landing/v2/HeroBlockPortrait.tsx`

Для каждого из 4 wrapper-`div` виджетов:
```
absolute
top-[Ymob]   left-[Xmob]/right-[Xmob]   w-[Wmob]
sm:top-[Ytab] sm:left-[Xtab]            sm:w-[Wtab]
lg:top-[Ydsk] lg:left-[Xdsk]            lg:w-[Wdsk]
```
Никаких `xl:` сдвигов и никаких `hidden sm:block`.

Контейнер портрета: фиксированная высота на каждом брейкпойнте (`h-[560px] sm:h-[600px] lg:h-[620px]`) — чтобы абсолютные координаты не «уезжали».

## Верификация
Playwright-скрипт под `/tmp/browser/hero-layout/` на 375/768/1280, скриншоты + проверка непересечения bbox всех 4 виджетов. Итерируем координаты до зелёного результата.
