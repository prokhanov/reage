
## Цель

Гарантировать, что структурные подзаголовки внутри разделов отчёта (Блок 3 «Интерпретация биомаркеров», Блок 4 «Общая оценка / Сильные стороны / Дефициты», подблок «Что это значит для вас:» внутри карточек биомаркеров) всегда приходят на фронт с префиксами `##`/`###`/`####`, даже если модель забыла их поставить.

Промпты уже исправлены — это второй слой защиты. Старые отчёты не трогаем: нормализация работает только на этапе валидации свежесгенерированного текста.

## Что делаем

В edge function `supabase/functions/report-qa/index.ts` добавляем функцию `normalizeStructuralHeadings(text)`:

1. **Идемпотентность.** Если строка уже начинается с `#`, `##`, `###`, `####` — пропускаем.
2. **Построчная проверка** после `translateEnglishFragments`, до сохранения.
3. **Whitelist канонических строк** (регистронезависимо, кавычки нормализуются):

   Уровень `##`:
   - `Интерпретация биомаркеров` + синонимы: `Разбор показателей`, `Разбор биомаркеров`, `Ключевые показатели`, `Ключевые биомаркеры`
   - `Общая оценка системы организма` (полная строка с любым названием в кавычках — matches по префиксу)

   Уровень `###`:
   - `Сильные стороны организма`
   - `Дефициты и дисфункции`

   Уровень `####`:
   - `Что это значит для вас:` (с двоеточием; вариант без двоеточия тоже подхватываем и добавляем)

4. **Названия биомаркеров НЕ трогаем.** Парсер (`stripLeadingBiomarkerName`) уже вырезает первую строку после якоря биомаркера и передаёт её как title карточки в `BiomarkerCard`. Если добавить `### `, будет дубль или сломается вырезание. Оставляем текущий поток.

5. **Категорийный заголовок Блока 1 НЕ трогаем** — парсер его тоже вырезает (`stripLeadingCategoryHeader`).

## Где вызывать

В `report-qa/index.ts`, в основном пайплайне валидации, для каждого раздела (recommendation с `type` из категорий) — после того как отработали `translateEnglishFragments` и `injectMissingBiomarkerAnchors`, но до `UPDATE recommendations`. Одна вставка вызова, без изменения архитектуры.

## Что НЕ меняем

- Промпты (уже поправлены).
- `parser.ts` — логика вырезания категорийного заголовка и имени биомаркера остаётся.
- `ProseMarkdown.tsx` — уже умеет рендерить `h2/h3/h4`.
- Старые сохранённые отчёты в БД — миграция не делается.
- Никаких «умных» эвристик по длине строки — только точный whitelist.

## Как проверить

1. Сгенерировать новый отчёт (можно через админский Analysis Wizard на тестовом пациенте).
2. В БД: `SELECT text FROM recommendations WHERE user_id = ... AND type = 'Сердечно-сосудистая система'` — убедиться, что канонические строки идут с префиксами `##`/`###`/`####`.
3. Открыть отчёт на фронте: подзаголовки визуально выделены как заголовки, а не как обычный текст.
4. Прогнать edge function повторно (report-qa идемпотентен) — двойных префиксов `## ## X` не появляется.

## Технические детали

- Файл: `supabase/functions/report-qa/index.ts` — одна новая функция + одна строка вызова в основном цикле.
- Работа по строкам (`text.split('\n').map(...).join('\n')`).
- Нормализация ключа сравнения: `.trim().toLowerCase().replace(/[«»"'`]/g, '"')`.
- Для «Общая оценка системы организма …» — match по префиксу строки (название категории в кавычках варьируется).
- Для «Что это значит для вас» — добавляем `:` если его нет.
- Возвращаем изменённый текст только если реально что-то поменяли (для чистого diff в логах).
