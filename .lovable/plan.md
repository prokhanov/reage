

# Обновление демо-данных в БД вместо дублирования маппинга

## Проблема

Демо-данные в БД используют 61 код биомаркеров, из которых 23 не совпадают с маппингом в `DEMO_TO_DB_CODE`. Из этих 23:
- **11 нужно переименовать** в коды, уже существующие в маппинге
- **5 нужно добавить** в маппинг (новые биомаркеры в БД, которых раньше не было)
- **7 нужно удалить** из демо-данных (нет в таблице `biomarkers` или исключены)

После исправления: 61 − 7 = **54 биомаркера** корректно отображаются.

## Решение

### 1. Обновить демо-данные в БД (SQL миграция)

Переименовать коды в `male_data->'biomarkers'` (и `female_data` когда будут добавлены):

| Текущий код | → Новый код | Почему |
|---|---|---|
| `GLUC` | `GLU` | Уже есть в маппинге → "Глюкоза" |
| `CREAT` | `CREA` | Уже есть → "Креатинин" |
| `LYM` | `LYMPH` | Уже есть → "Лимфоциты" |
| `NEU` | `NEUT` | Уже есть → "Нейтрофилы" |
| `TBIL` | `BIL` | Уже есть → "Билирубин" |
| `TPROT` | `TP` | Уже есть → "Общий белок" |
| `TESTOST` | `TEST` | Уже есть → "Тестостерон общий" |
| `HOMOCYS` | `HCY` | Уже есть → "Гомоцистеин" |
| `D3` | `VITD` | Уже есть → "25-ОН витамин D" |
| `CA` (кальций) | `CACL` | Уже есть → "Кальций (Ca)" |
| `ATHERIX` | `CA` | Маппинг CA → "Индекс атерогенности" |

Удалить из демо-данных коды без соответствия в БД:
- `DBIL`, `FIB`, `GFR`, `P`, `URICACID` — нет в таблице biomarkers
- `IGA`, `TIBC` — исключены из системы

### 2. Добавить 5 новых кодов в `DEMO_TO_DB_CODE` (`src/lib/biomarkerCodeMap.ts`)

```
ALB → "Альбумин"
B12 → "Витамин B12"
HOMAIR → "HOMA-IR (расчётный)"
MG → "Магний (сыворотка)"
ZN → "Цинк"
```

### 3. Удалить исключённые коды из маппинга (`src/lib/biomarkerCodeMap.ts`)

Убрать: `RETIC`, `HAPTO`, `TROP`, `MYO`, `IGA`, `IGE`, `TIBC`, `TSAT`, `CD19`

### 4. Вернуть подсчёт маркеров (`src/pages/Analyses.tsx`)

Строка 152: вернуть `biomarkers_count` с фильтрацией по `DEMO_TO_DB_CODE`:
```typescript
biomarkers_count: demoData.biomarkers
  .filter((b: any) => (b.analysis_index || 0) === index)
  .filter((b: any) => DEMO_TO_DB_CODE[b.code] !== undefined)
  .length,
```

### 5. Показать заголовок с количеством в демо (`src/pages/AnalysisDetail.tsx`)

Строка 515: для демо-анализов показывать заголовок "Введенные значения (N)" без табов (без переключателя на "Все показатели").

## Итог

- Демо-данные в БД используют те же коды что и маппинг — нет дублирования
- 54 биомаркера корректно отображаются (вместо 39)
- Количество маркеров совпадает на странице списка и внутри анализа
