

# Несоответствие количества маркеров: 61 vs 39

## Проблема

На странице "История анализов" отображается "61 маркеров", а при открытии анализа — "Введенные значения (39)" и "Все показатели (39)".

**Две причины:**

1. **Список анализов** (`Analyses.tsx`, строка 152) считает ВСЕ записи в `demoData.biomarkers` для данного `analysis_index` — включая коды, для которых нет соответствия в базе данных (через `DEMO_TO_DB_CODE`). Это даёт 61.

2. **Детальная страница** (`AnalysisDetail.tsx`, строка 159-189) фильтрует биомаркеры: если код не найден в базе (`metadataMap.get(dbCode)` возвращает null), запись отбрасывается. Остаётся 39.

**Дополнительная проблема:** Вкладка "Все показатели" в демо-режиме показывает те же 39, что и "Введенные значения", потому что `biomarkers` заполняется только из `demoValues` (строка 194), а не из полного списка биомаркеров в базе.

## Решение

### 1. `src/pages/Analyses.tsx` — считать только маппящиеся маркеры

Строка 152: вместо простого `.filter((b) => (b.analysis_index || 0) === index).length` добавить проверку маппинга кода через `DEMO_TO_DB_CODE`, чтобы считались только биомаркеры с валидным кодом в БД.

```typescript
biomarkers_count: demoData.biomarkers
  .filter((b: any) => (b.analysis_index || 0) === index)
  .filter((b: any) => {
    const dbCode = DEMO_TO_DB_CODE[b.code] || b.code;
    return dbCode !== undefined;
  }).length,
```

Но это всё равно может не совпасть, потому что часть кодов есть в маппинге, но отсутствует в таблице `biomarkers`. Более надёжный вариант — просто убрать `biomarkers_count` из карточки демо-анализа или показывать его после реальной загрузки метаданных.

**Оптимальное решение:** На странице списка анализов в демо-режиме не показывать счётчик маркеров вовсе, либо поставить тот же механизм фильтрации что и на детальной странице. Самый простой путь — не показывать badge с количеством маркеров для демо-анализов, так как точный подсчёт требует запроса в БД.

### 2. `src/pages/AnalysisDetail.tsx` — вкладка "Все показатели" в демо-режиме

Строка 194: в демо-режиме `biomarkers` заполняется только из `demoValues`, поэтому обе вкладки показывают одно и то же. Два варианта:

**Вариант A (простой):** Скрыть вкладку "Все показатели" в демо-режиме, оставив только "Введенные значения". Логично, потому что в демо все значения уже введены.

**Вариант B:** Загружать полный список биомаркеров из БД даже в демо-режиме (как для реальных анализов). Это покажет ~79 биомаркеров во вкладке "Все показатели", из которых 39 с значениями.

**Рекомендую вариант A** — скрыть вкладку "Все показатели" для демо-анализов. Она не несёт смысла если все значения уже показаны.

### Конкретные изменения

**`src/pages/Analyses.tsx`:**
- Импортировать `DEMO_TO_DB_CODE` из `@/lib/biomarkerCodeMap`
- Строка 152: отфильтровать по наличию маппинга, или убрать badge для демо

**`src/pages/AnalysisDetail.tsx`:**
- Строки 514-522: скрыть TabsList и показывать только контент "Введенные значения" когда `isDemoAnalysis === true`
