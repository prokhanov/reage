## Что я обнаружил при анализе архитектуры

### 1. Pipeline генерации «Назначений» (edge function `analyze-biomarkers`)

ИИ через JSON tool-calling возвращает блок prescription со структурой:
```
{
  nutraceuticals: [...],            // → таблица prescriptions (9 строк у Дарбинян)
  lifestyle: { nutrition, activity, sleep },  // → recommendations.content_json
  follow_ups: [...]                 // → recommendations.content_json
}
```
Затем edge-функция должна создать ОДНУ запись в `recommendations` с `type='Назначения'` и `content_json={lifestyle, follow_ups, ...}`.

### 2. Состояние БД (проверено напрямую)

- У Дарбинян (`d8d632d4-...`) **есть** свежая запись `recommendations.type='Назначения'` от 25.04.2026 20:31, и `content_json` содержит и `lifestyle` (4 пункта в питании, 3 в активности, 3 в сне), и `follow_ups` (5 специалистов). **Данные на месте.**
- Во всей таблице `recommendations` это **единственная** запись с типом `'Назначения'` — у других пациентов её нет вообще. Значит для всех старых отчётов блок не сгенерируется никогда без перезапуска.

### 3. Куда эти данные попадают в UI

| Место | Источник lifestyle/follow_ups | Статус |
|---|---|---|
| `/recommendations` модалка отчёта | `grouped["Назначения"][0].content_json` | Код корректный, должен рендерить |
| `/recommendations` экспорт PDF | то же самое | Код корректный |
| `/prescriptions` страница списка | отдельный `useQuery` по type='Назначения' | Я добавил, но переместил блок наверх (вы не просили) |

### 4. Гипотезы почему вы не видите блоки

A. **React Query кэш** не инвалидируется после новых тестов: ключ `["recommendations-advisory", userId, demoMode]` стабилен → стейл-данные `null`.
B. **Конфликт со snapshot-рендером**: `summary.content_json` содержит ReportSnapshot, при невалидной парсинге снапшота фолбек на категории работает, но это не должно затрагивать prescription-блок. Нужно проверить условие `hasPrescriptionsBlock` в момент открытия модалки.
C. **Условие `enabled: !demoMode`** в advisory-query может ложно срабатывать, если у админа demo включен и он выходит/входит во View As — стейл флага.
D. Для других пациентов блока **физически нет в БД** — pipeline должен прогнаться заново через «Создать отчёт».
