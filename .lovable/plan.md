

## Исправление визуализации открытых диапазонов

### Проблемы найдены в 3 файлах

#### 1. `BiomarkerRangeBar.tsx` --- основная проблема

Логика построения boundaries сломана для открытых диапазонов:

- **optMin=NULL** (ниже = лучше, например HbA1c): строка 71-73 вставляет второй boundary на `scaleMin` с цветом "optimal" ПОСЛЕ уже существующих boundaries normMin/critMin. Это нарушает порядок слева направо и ломает сегменты.
- **optMax=NULL** (выше = лучше, например тестостерон): строки 80-83 просто ничего не делают ("don't add boundary"), но дальше на строке 85-87 normMax добавляется с цветом "risk". Визуально optimal зона заканчивается на normMax --- это допустимо, но нет acceptable зоны между optimal и normal.

**Решение**: полностью переписать логику построения сегментов. Вместо "добавлять boundaries по порядку" --- собрать все точки, отсортировать, и для каждого интервала определить цвет по принципу "в какую зону попадает середина интервала". Это надежно работает для любых комбинаций NULL.

#### 2. `Trends.tsx` --- график трендов

Строка 472: `trendData[0].refMin != null && trendData[0].refMax != null` --- если одна из границ нормы NULL, зоны вообще не рисуются. Нужно: `refMin != null || refMax != null`.

Строка 477: `if (d.optimalMin != null && d.optimalMax != null)` --- для открытых оптимумов зеленая зона не рисуется. Нужно обрабатывать односторонние optimal ranges, используя ось графика как границу.

Строки 484-488: acceptable зоны тоже не рисуются если один из optimal = null.

#### 3. `AnalysisDetail.tsx` --- карточки биомаркеров

Строка 745: `{min !== null && max !== null && (` --- скрывает статус-бейдж если одна из границ null. Нужно: `min !== null || max !== null`.

Строка 765: аналогично скрывает блок референсных значений.

### Файлы для изменения

1. **`src/components/BiomarkerRangeBar.tsx`** --- переписать segment-building логику для корректной работы с NULL границами
2. **`src/pages/Trends.tsx`** --- обновить условия рисования ReferenceArea зон для поддержки открытых диапазонов
3. **`src/pages/AnalysisDetail.tsx`** --- заменить `&&` на `||` в условиях отображения статуса и референсных значений
