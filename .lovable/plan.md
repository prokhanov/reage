## Что делаем

Включаем режим работы «по звонку оператора» с возможностью переключения между онлайн-записью и записью по телефону. Все тексты статусов баннера редактируются админом. На клиенте — упрощённое окно «Мы вам перезвоним» с предзаполненным номером пользователя.

## 1. База данных (миграция)

Новая таблица `booking_mode_settings` (singleton, 1 строка):

- `mode` — `'online' | 'phone'` (по умолчанию `'phone'`)
- `phone_status_texts` — `jsonb` (заголовок + подзаголовок для каждого статуса в режиме «по звонку»)
- `online_status_texts` — `jsonb` (то же для онлайн-режима)

Дефолтные тексты для режима **phone**:
- `waiting_call` — «Ожидайте звонка менеджера» / «Мы свяжемся с вами в ближайшее время для согласования даты»
- `no_answer` — «Не дозвонились» / «Перезвоним ещё раз. Вы также можете запросить повторный звонок»
- `scheduled` — «Ожидайте визита специалиста» / `{date} в {time} • {address}`
- `received` — «Ваши анализы получены!» / «Скоро результаты появятся. Обычно это занимает 5 дней»
- `collected` — «Анализы обрабатываются» / «Результаты скоро появятся в вашем профиле»

Для режима **online** — текущие тексты из `AnalysisBookingBanner.tsx`.

Расширяем допустимые значения `analysis_bookings.status`: добавляем `'waiting_call'` и `'no_answer'` (поле уже `text`, ограничения нет — менять схему не нужно, только код).

RLS: read для `authenticated` и `anon` (нужно баннеру), write только для `superadmin`.

## 2. Админ — настройки в «Запись на анализы»

В `src/pages/admin/AnalysisBookings.tsx` добавляем новую вкладку **«Настройки»** рядом с существующими (Tabs уже есть в файле).

Содержимое:
- Switch «Онлайн-запись / По звонку оператора» — переключает `mode`.
- Два блока редактирования текстов (раскрываются по выбранному режиму или показываются оба под аккордеоном): для каждого статуса — два инпута (заголовок и подзаголовок).
- Кнопка «Сохранить».

Хук `useBookingModeSettings` — read + invalidate после mutation.

## 3. Клиент — баннер и диалог

### `AnalysisBookingBanner.tsx`
- Подгружает `booking_mode_settings`.
- Если bookings нет — статус по умолчанию: `waiting_call` (в режиме phone) или прежний «Запишитесь на анализы» (в режиме online).
- Все заголовки/подзаголовки берутся из настроек по `status`.
- Кнопка «Назначить дату» в режиме phone открывает новый компактный диалог `CallbackRequestDialog` вместо `AnalysisBookingDialog`.
- Кнопка скрывается на статусах `received` / `collected` (как сейчас) и `waiting_call` оставляем кнопку «Запросить звонок снова» — опционально, на этом этапе достаточно скрывать кнопку, если статус `waiting_call`. На `no_answer` показываем кнопку «Запросить звонок».

### `CallbackRequestDialog.tsx` (новый)
- Заголовок: «Мы вам перезвоним».
- Текст: «Менеджер свяжется с вами для согласования даты и адреса визита».
- Поле «Телефон» — предзаполнено из `profiles.phone`, доступно для редактирования.
- Кнопка «Подтвердить» → создаёт/обновляет запись в `analysis_bookings` со статусом `waiting_call` (без `booking_date/time/address`), обновляет `profiles.phone` если изменился.
- Кнопка «Отмена».

### Админ — управление статусом в существующей таблице бронирований
В `statusLabels` / select статуса бронирования добавляем `waiting_call` и `no_answer`, чтобы оператор мог переключать вручную после звонка.
