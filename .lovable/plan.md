
# План: Очистка назначений из базы данных

## Текущая ситуация

В базе данных обнаружено **47 назначений (prescriptions)** со статусом `on_review` для одного пользователя — все они связаны с одним анализом. Эти записи, вероятно, были созданы при повторных запусках AI-генерации.

## Решение

Создам edge function `cleanup-prescriptions`, которая удалит все назначения указанного пользователя (или все назначения со статусом `on_review`, если нужно).

## Технические детали

### Шаг 1: Создание edge function

Создаю функцию `supabase/functions/cleanup-prescriptions/index.ts`:

```text
Функция принимает параметры:
├─ user_id (опционально) — удалить назначения конкретного пользователя
├─ analysis_id (опционально) — удалить назначения конкретного анализа  
└─ status (опционально) — удалить только назначения с определённым статусом

Защита:
├─ Требуется авторизация (Bearer token)
└─ Только superadmin может вызывать функцию
```

### Шаг 2: Вызов функции

После создания и деплоя вызову функцию для удаления всех 47 записей:

```
POST /cleanup-prescriptions
Body: { "user_id": "e47bc111-0064-4e03-8f0f-0ae554252948" }
```

### Шаг 3: Проверка

Проверю, что записи удалены:
- Таблица `prescriptions` для этого пользователя пуста
- Страница назначений в UI не показывает старые данные

## Результат

После выполнения:
- Все 47 дублирующих назначений будут удалены
- База данных будет очищена от «мусорных» записей
- Можно будет сгенерировать новые назначения корректно
