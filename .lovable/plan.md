

# Проблема: назначения не видят находки из разделов

## Корневая причина

В коде на **строке 989-991**:
```js
const keyFindings = Object.entries(categoryReports)
  .map(([category, report]) => `${category}: ${report.substring(0, 300)}...`)
  .join('\n\n');
```

Назначения получают **только 300 символов** от каждого раздела. Это 2-3 предложения — AI, генерирующий назначения, физически не видит дефициты и рекомендации, которые подробно описаны в середине/конце каждого отчёта (который может быть 10000+ символов).

Для сравнения, общее резюме получает **5000 символов** на категорию (строка 800).

## Что меняем

### 1. Увеличить `keyFindings` с 300 до 3000 символов на категорию

Строка 990 в `analyze-biomarkers/index.ts`:
```js
// Было: report.substring(0, 300)
// Станет: report.substring(0, 3000)
```

Это даст AI-генератору назначений в 10 раз больше контекста из каждого раздела — он увидит конкретные дефициты, описания рисков и рекомендации по добавкам.

### 2. Добавить извлечение рекомендательных секций из отчётов

Дополнительно к `keyFindings`, извлекать из каждого отчёта секцию с рекомендациями (ищем по якорю `anchor:recommendations_start`) и передавать их отдельным блоком в промпт назначений. Это гарантирует, что конкретные рекомендации из разделов (дозировки, добавки) попадут в назначения.

### 3. Добавить плейсхолдер `{categoryRecommendations}` в промпт `prescriptions_user`

Обновить промпт в БД, добавив новый блок:
```
РЕКОМЕНДАЦИИ ИЗ ДЕТАЛЬНЫХ РАЗДЕЛОВ (ОБЯЗАТЕЛЬНО УЧЕСТЬ):
{categoryRecommendations}

КРИТИЧНО: Каждая рекомендация из детальных разделов ДОЛЖНА быть отражена 
в назначениях. Если в разделе упомянут дефицит и рекомендована добавка — 
она ОБЯЗАНА появиться в списке назначений.
```

## Объём работ
- 1 изменение в edge function (увеличить лимит + извлечение рекомендаций)
- 1 UPDATE в `ai_prompt_settings` (`prescriptions_user`)
