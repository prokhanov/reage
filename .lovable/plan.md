## Проблема
Мок-генератор в мастере создания анализа (`AnalysisStep1.tsx`) заполняет все биомаркеры из `biomarkers`, не учитывая тариф. Админу нужна возможность выбрать тариф, чтобы мок повторял реальную ситуацию пациента на этом тарифе (basic — 83 маркера, plus — 110, expert — 115).

## Решение
Добавить выбор тарифа в диалоге генерации мока. Селектор тарифа + уровень здоровья — оба задаются админом перед генерацией.

### Изменения (только `src/components/admin/AnalysisStep1.tsx`)

1. **Диалог `showHealthDialog`** — добавить в шапку `<Select>` со списком тарифов, подгружаемых из `subscription_plans` (name + biomarker count в подписи опции, напр. «Plus (110 маркеров)»).
   - Дефолт: активный тариф пациента из `subscriptions` (status=active). Если подписки нет — первый тариф в алфавитном порядке.
   - Состояние: `selectedPlanId: string | null`.

2. **Загрузка списка тарифов** — при первом открытии диалога параллельно тянуть:
   - `subscription_plans` (id, name),
   - счётчик `plan_biomarkers` per plan (одним запросом с `count`, либо агрегатом),
   - активную подписку пациента для дефолта.
   Кэшировать в `useState`, чтобы не дёргать повторно.

3. **`handleGenerateMock(healthLevel)`** — принимает уже выбранный `selectedPlanId`:
   - Дополнительно грузить `plan_biomarkers` для выбранного плана → `Set<biomarker_id>`.
   - В цикле пропускать `bm`, которого нет в `Set` (расчётные по-прежнему считаются формулой в конце).
   - После генерации derived — фильтровать их тем же `Set` (если расчётный маркер не входит в тариф — не добавляем).

4. **Toast об успехе** — «N значений (тариф Plus, уровень Здоровый)».

5. **UI** — тариф отображается в диалоге как обязательный шаг перед кнопками уровней здоровья. Кнопки уровней остаются как есть; клик по кнопке = «сгенерировать для выбранного тарифа + этот уровень».

## Что НЕ трогаем
- Логику зон/весов, формулы расчётных маркеров.
- Авто-импорт PDF.
- Схему БД, RLS, edge-функции.
- Реальную подписку пациента — выбранный в моке тариф на неё не влияет.
