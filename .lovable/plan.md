## Проблема
В карточке «Операции и процедуры» перечень «Нет: переливание крови…, донор…, вакцинация…» выглядит как непонятный технический дамп в одной строке. Пользователь не понимает, что это ответы «Нет» на подпункты анкеты.

## Решение (только UI, `src/components/profile/MedicalAnketaCard.tsx`)

1. Показывать каждый пункт отдельной строкой (как «Да»‑пункты), с бейджем «Нет» в мьютед‑стиле — единый формат для всех вопросов.
2. Убрать сводную строку «Нет: …».
3. Порядок: сначала положительные ответы (бейдж «Да», rose), затем отрицательные (бейдж «Нет», muted/outline). Детали операции (`surgery_year_details`) оставить под соответствующей строкой.
4. Для мобильной верстки: `flex items-start justify-between gap-3`, лейбл переносится, бейдж фиксированной ширины справа.

Итого пользователь видит понятный список:
- Операции за последний год — Да · Аппендикс
- Переливание крови за последние 3 мес. — Нет
- Сдавали кровь как донор < 3 мес. — Нет
- Вакцинация за последние 2 недели — Нет

Изменения только презентационные, данные и схема БД не трогаем.
