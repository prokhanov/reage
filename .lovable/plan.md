
# Mobile redesign — страница «Моё здоровье» (`/dashboard`)

Ниже — что я увидел на скриншотах мобильной версии (390×844) и что предлагаю исправить, чтобы интерфейс выглядел как качественное мобильное приложение с единым гайдлайном.

## Найденные проблемы

1. **Прозрачная «шапка» (mobile header)** — `bg-secondary/90 backdrop-blur` всё равно просвечивает: при скролле сквозь неё видно H1 и заголовки таблиц. Контент «налезает» на хедер. См. shot_01–shot_12.
2. **Биомаркерные таблицы выходят за экран** — числа и единицы обрезаны справа («32.1…», «480.…», «13.94 п…», «4.54 ×10¹…»). Горизонтального скролла визуально нет, пользователь не понимает, что есть скрытый контент.
3. **Огромная вертикальная высота строк биомаркеров** (~140 px на строку). Список выглядит «рыхлым» и требует бесконечного скролла.
4. **Заголовок «Добро пожаловать, …»** — слишком большой, в градиенте, плохо читается, отступы не согласованы с карточками выше.
5. **Карточки‑баннеры сверху** («Ожидайте звонка менеджера», «Для записи на первый анализ…») имеют разную плотность, толщину обводки и радиус, чем остальные карточки страницы.
6. **Сетка статистики 2×2** (Индекс здоровья / Анализов / Последнее изменение / Общий прогресс / Следующий анализ) — карточки разной высоты, контент по‑разному выровнен, кнопка «Назначить внеурочный» переносится некрасиво.
7. **Табы «Текущее состояние / Динамика» и «Маркеры / Тренды»** — фон полупрозрачный, при скролле сливаются с содержимым; одна группа табов вообще оказывается «под» шапкой.
8. **Карточка «Место среди ровесников»** — три колонки «Топ 40 % / −3.0 / лет» прижаты друг к другу, иконка трофея слева оторвана.
9. **Карточка «Текущий вес» / «ИМТ»** — крупная цифра прижата к левому краю, иконка справа, кнопки `Добавить` / `История` другой ширины и формы, чем кнопки в других карточках.
10. **Несогласованные радиусы (12 / 16 / 24 px) и паддинги (16 / 20 / 24 px)** между секциями. Нет единой шкалы spacing.
11. **Числа** на странице используют то моноширинный шрифт, то sans‑serif, то градиент. Цвета статусов биомаркеров тоже выглядят кислотно на тёмном фоне.
12. **Аккордеоны категорий биомаркеров** — заголовок («Энергия и восстановление») переносится в 2 строки, бейдж «19» и прогресс‑бар прыгают в высоту.

## План исправлений

### 1. Mobile header (DashboardLayout)
- Сделать шапку полностью непрозрачной (`bg-background` + тонкая нижняя граница, без `backdrop-blur`), чтобы контент не просвечивал.
- Уменьшить высоту: 56 px вместо 64 px, иконка‑меню 40×40 тач‑таргет, лого по центру.
- Контент `main` отступает от шапки ровно на её высоту (`pt-14`), чтобы H1 не «нырял» под шапку.

### 2. Единая дизайн‑система для мобайла
В `index.css` / Tailwind токенах закрепить и применить ко всей странице:
- Радиусы: только `rounded-2xl` (карточки) и `rounded-xl` (кнопки/чипы/инпуты).
- Spacing: внешний паддинг страницы `px-4`, между секциями `space-y-4`, внутри карточки `p-4` (мобайл) / `p-6` (≥sm).
- Типографика: H1 `text-2xl font-semibold`, H2 `text-lg`, метрики `text-4xl font-semibold tabular-nums` (один и тот же класс на все крупные цифры — вес, ИМТ, био‑возраст, индекс здоровья).
- Цвета статусов биомаркеров — через семантические токены (`--status-optimal`, `--status-warning`, `--status-alert`), приглушённые тона для тёмной темы.

### 3. Верхние баннеры
- Привести «Ожидайте звонка» и «Заполнить паспортные» к одному компоненту `InfoCard` с одинаковыми отступами, иконкой 40×40 в круге, единым CTA‑кнопкой во всю ширину карточки на мобайле.
- Кнопка‑крестик «закрыть» — 40×40 тач‑таргет.

### 4. Заголовок и блок «Биологический возраст»
- H1 без градиента, обычный `foreground`. Подпись «Паспортный возраст: 37 лет» под ним мелким `text-muted-foreground`.
- Карточку «Ваш биологический возраст» сделать выше по визуальной иерархии: круг с возрастом + чип «−3.0 лет моложе» под ним. Таб‑группа `Текущее / Динамика` — `Tabs` shadcn с непрозрачным фоном.

### 5. Сетка KPI
- Перевести «Индекс здоровья / Анализов / Последнее изменение / Общий прогресс / Следующий анализ» в адаптивную сетку:
  - Mobile: 1 большая карточка «Индекс здоровья» + сетка 2×2 одинакового размера для остальных, фиксированная высота, контент выровнен `flex flex-col justify-between`.
- Кнопку «Назначить внеурочный» вынести из карточки в отдельный full‑width CTA под блоком, либо ужать до иконки `+`.

### 6. Карточки «Вес» и «ИМТ»
- Единый шаблон: верх — название + иконка справа в круге 40×40; центр — крупная цифра + единица; низ — тренд/статус; кнопки `Добавить` / `История` равной ширины (`grid grid-cols-2 gap-2`).

### 7. Аккордеоны категорий биомаркеров
- Заголовок в одну строку (`truncate`), бейдж и прогресс‑бар выровнены справа в отдельном блоке; на узком экране прогресс‑бар уходит на вторую строку под заголовком, а не растягивает шапку.

### 8. Список биомаркеров (главная проблема)
Заменить таблицу на **карточно‑списочный layout** для `<sm`:
```
┌───────────────────────────────────┐
│ Альбумин                  32.1 ▮▮ │  ← название + значение в одной строке
│ ALB · г/л                  норма  │  ← код + единица + статус‑чип
└───────────────────────────────────┘
```
- Высота строки ~64 px (вместо 140).
- Значение выровнено по правому краю, `tabular-nums`, единица измерения мелким текстом рядом.
- Статус‑чип цветной (зелёный/жёлтый/оранжевый/красный) вместо подсветки фона всей строки.
- На `≥md` остаётся текущая таблица.

### 9. Табы внутри страницы
- `Tabs` shadcn с непрозрачным фоном `bg-muted`, sticky‑позиционирование только если действительно нужно; иначе обычный inline блок.
- Активный таб — солид‑фон, без градиента, чтобы не конкурировал с CTA‑кнопками.

### 10. Прочее
- Везде где есть горизонтальный скролл (таблицы при `md`+), добавить `overflow-x-auto` и тень‑индикатор справа, чтобы было видно, что можно скроллить.
- Проверить `safe-area-inset-bottom` для устройств с домашней полосой.

## Технические детали

Затрагиваемые файлы:
- `src/components/DashboardLayout.tsx` — шапка, отступы.
- `src/pages/Dashboard.tsx` — сетка KPI, заголовок, баннеры.
- `src/components/AnalysisBookingBanner.tsx` — унификация баннеров.
- `src/components/dashboard/HealthPercentileCard.tsx`, `NextAnalysisCard.tsx`, `SystemRatingsCard.tsx` — единый шаблон карточек.
- Компонент(ы) рендера биомаркеров (найду точное имя в `src/components` — вероятно `BiomarkersTable` / `CategoryAccordion`) — добавить mobile‑карточный режим.
- `src/index.css` / `tailwind.config.ts` — токены радиусов и статус‑цветов, если их ещё нет.

Все правки — только UI/презентация, без изменения бизнес‑логики и данных.

## Что НЕ входит

- Не трогаю расчёты био‑возраста, индекса здоровья, источники данных.
- Не меняю порядок секций (порядок остаётся как у вас сейчас).
- Десктоп‑верстка не меняется — все изменения под mobile breakpoint, на `md+` всё как было.
