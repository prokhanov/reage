## Что именно сломано

1. **Регистрация по пригласительной ссылке**
   - Backend-триггер при создании пользователя уже создаёт запись в `profiles`.
   - Edge Function `register-staff` потом повторно делает `insert` в `profiles` с тем же `id`.
   - Итог: `duplicate key value violates unique constraint "profiles_pkey"`, а пользователю показывается «Сервис временно недоступен».

2. **Копирование пригласительной ссылки**
   - В `UserManagement.tsx` перед копированием выполняется асинхронная проверка токена в базе.
   - После `await` браузер может считать, что действие уже не связано напрямую с кликом пользователя, и блокировать clipboard.
   - Поэтому toast может показывать «скопировано», но буфер реально не обновляется.

## План исправления

### 1. Исправить `register-staff`
- Заменить повторный `insert` профиля на безопасный `upsert`/`update` существующего профиля.
- Передавать в `createUser` полный metadata-набор: `first_name`, `last_name`, `birth_date`, `gender`, чтобы основной backend-триггер сразу создал корректный профиль.
- После создания пользователя обновлять профиль идемпотентно, без падения при уже существующей записи.
- Для staff-регистрации убрать автоматическую пациентскую роль, которую даёт общий signup-триггер, и оставить только выбранную административную кастомную роль.
- Не включать `auto_confirm_email=true`; оставить проектное правило про письмо подтверждения без блокировки доступа.
- Возвращать пользователю нормальные русские ошибки, а не общий «Сервис временно недоступен» там, где причина известна.

### 2. Исправить копирование ссылки
- Убрать async-проверку токена из прямого обработчика кнопки «Копировать».
- Копировать уже известную ссылку синхронно от клика пользователя: `copyToClipboard(fullUrl)` сразу в обработчике.
- Проверку валидности/перегенерацию оставить отдельной кнопкой/действием «Проверить» или текущим глазом/поповером, но не перед copy.
- В окне создания пользователя не обещать автоматическое копирование после async-создания как единственный сценарий: показывать ссылку в поле и давать отдельную кнопку копирования, которая вызывается прямым кликом.
- Если браузер всё равно блокирует clipboard, показывать ссылку в поле с выделением для ручного копирования.

### 3. Провести реальные проверки
- Локально проверить кодовые сценарии:
  - создание invite-ссылки;
  - клик «Копировать» без предварительного await;
  - открытие `/register-staff?invite=...`;
  - отправка формы регистрации.
- После правок задеплоить изменённую Edge Function `register-staff` и протестировать её через прямой вызов функции/логи.
- Проверить свежие логи `register-staff`: ошибка `profiles_pkey` должна исчезнуть.

## Файлы, которые будут изменены

- `supabase/functions/register-staff/index.ts`
- `src/pages/admin/UserManagement.tsx`
- `src/components/admin/CreateUserDialog.tsx`
- при необходимости минимально `src/lib/copyToClipboard.ts`, если понадобится усилить fallback/проверку результата

## Ожидаемый результат

- Пригласительная ссылка реально попадает в буфер по кнопке копирования.
- Регистрация по ссылке больше не падает на создании профиля.
- Новый сотрудник получает staff-доступ по приглашённой роли, без лишней пациентской роли от общего signup-триггера.
