# Telegram-бот уведомлений для админов

Бот будет отправлять в Telegram-чат уведомления о ключевых событиях. На старте — два события: **новая регистрация** и **новая оплата**. Архитектура заложена так, чтобы позже легко добавить остальные (новый анализ, новая бронь и т.п.).

## Что увидит пользователь

В админ-меню (сайдбар) появится новый пункт **«Telegram уведомления»** (раздел доступен только superadmin). Внутри — одна страница с тремя блоками:

### Блок 1. Подключение бота
- Поле **Bot Token** (хранится как секрет на сервере, в UI показывается маскированно: `1234…abcd`).
- Поле **Chat ID** (числовой ID чата/группы/канала, куда слать уведомления).
- Кнопка **«Сохранить»**.
- Кнопка **«Проверить подключение»** — дергает Telegram `getMe`, показывает имя бота и статус.
- Индикатор статуса: 🟢 подключен / 🔴 не настроен / 🟡 ошибка (с текстом ошибки).
- Сворачиваемая **пошаговая инструкция «для чайника»** (см. ниже).

### Блок 2. События для уведомлений
Список переключателей (включить/выключить каждый тип):
- ✅ Новая регистрация пациента
- ✅ Новая оплата подписки
- ⏸ (заглушки на будущее, выключены: новый анализ, новая бронь, отмена подписки и т.д.)

Для каждого включённого события — кнопка **«Отправить тестовое»**, которая шлёт в чат пример сообщения с пометкой `[ТЕСТ]`.

### Блок 3. Журнал отправок (последние 50)
Таблица: время · событие · статус (ок/ошибка) · текст ошибки. Помогает быстро понять, что бот молчит из-за неверного Chat ID, заблокированного бота и т.п.

## Формат сообщений в Telegram

**Новая регистрация** (HTML parse_mode):
```
🆕 <b>Новая регистрация</b>
👤 Имя Фамилия
📧 email@example.com
📱 +7 999 123-45-67
🕒 06.06.2026 14:32
```

**Новая оплата**:
```
💰 <b>Новая оплата</b>
👤 Имя Фамилия (email)
📦 Тариф: Премиум (годовой)
💵 120 000 ₽
🕒 06.06.2026 14:32
```

## Пошаговая инструкция для чайника (показывается в админке)

1. Открой в Telegram чат с **@BotFather**, отправь `/newbot`.
2. Введи имя бота (например, `ReAge Notifications`) и username (`reage_notify_bot`).
3. BotFather пришлёт строку вида `1234567890:ABCdefGhIJKlmNoPQRsTUvwxyz`. Это **Bot Token** — скопируй его.
4. Создай Telegram-группу (или используй личный чат). Добавь туда бота как участника. Если это группа — выдай боту права админа (иначе он не сможет писать).
5. Отправь в этот чат любое сообщение (например, «привет»).
6. Открой в браузере: `https://api.telegram.org/bot<ТВОЙ_ТОКЕН>/getUpdates` — найди в ответе `"chat":{"id": <число>}`. Это **Chat ID** (для групп он отрицательный, например `-1001234567890`).
7. Вставь Token и Chat ID в форму выше → «Сохранить» → «Проверить подключение». Должно прийти сообщение «✅ Бот подключен».
8. Нажми «Отправить тестовое» рядом с событием — убедись, что в чате появилось тестовое уведомление.
9. Готово — теперь при реальной регистрации/оплате уведомления будут приходить автоматически.

## Архитектура (техническая часть)

### Хранилище

Новая таблица `telegram_notification_settings` (одна строка-синглтон):
- `bot_token_secret_name` (ссылка на секрет; сам токен — в Supabase Secrets как `TELEGRAM_BOT_TOKEN`)
- `chat_id` (text)
- `enabled_events` (jsonb: `{"user_registered": true, "subscription_paid": true, ...}`)
- `is_active` (bool)
- `updated_by`, `updated_at`

Новая таблица `telegram_notification_log`:
- `event_type`, `payload` (jsonb), `status` (`sent|failed`), `error`, `sent_at`.
- RLS: только superadmin может читать; insert — только service_role.

RLS на settings: read/update только для superadmin.

### Edge-функции

1. **`telegram-notify`** (внутренняя, вызывается из других функций/триггеров)
   - Вход: `{ event_type, payload }`.
   - Читает settings, проверяет `is_active` и `enabled_events[event_type]`.
   - Форматирует сообщение по шаблону для event_type.
   - Шлёт через `https://api.telegram.org/bot<TOKEN>/sendMessage` (токен из `TELEGRAM_BOT_TOKEN` secret).
   - Пишет результат в `telegram_notification_log`.
   - Никогда не бросает ошибку наружу (не должна ломать регистрацию/оплату).

2. **`telegram-settings`** (для админ-UI)
   - `POST /test-connection` → `getMe` + опционально `sendMessage` «✅ Подключено».
   - `POST /test-event` → шлёт тестовое сообщение конкретного типа.
   - `POST /save` → обновляет settings и secret `TELEGRAM_BOT_TOKEN`.
   - Все ручки проверяют `has_role(auth.uid(), 'superadmin')`.

### Триггеры событий

- **Новая регистрация**: расширяем существующий триггер `handle_new_user` (или добавляем второй AFTER INSERT на `auth.users`), который через `pg_net` дергает edge-функцию `telegram-notify` с `event_type='user_registered'`. Альтернатива (проще и надёжнее): вызывать `telegram-notify` прямо из фронта после успешного `supabase.auth.signUp` — но триггер на БД лучше, т.к. не зависит от клиента.
- **Новая оплата**: триггер AFTER UPDATE на `public.subscriptions` при переходе `status` в `active` (или INSERT со `status='active'`). Тоже через `pg_net` → `telegram-notify`.

Для `pg_net` потребуется включить расширение и хранить URL edge-функции + service-role-ключ в SQL-функции (или в `vault`). Это стандартный паттерн Lovable Cloud.

### Безопасность

- Bot token хранится **только** в Supabase Secrets, в БД лежит только маркер «секрет задан». В UI отображается маскированно.
- `telegram-notify` использует `verify_jwt = false` + проверяет внутренний shared-secret в заголовке (чтобы извне нельзя было слать спам в чат).
- Все админ-ручки требуют superadmin-роль.
- Логи отправок не содержат токен.

### Расширяемость

Чтобы добавить новое событие в будущем:
1. Дописать шаблон сообщения в `telegram-notify` (одна функция-форматер).
2. Добавить ключ в `enabled_events` и чекбокс в UI (UI читает список событий из константы).
3. Подключить источник события (триггер БД или вызов из edge-функции).

## Файлы, которые появятся

- Миграция: 2 таблицы + RLS + триггеры на `auth.users` и `public.subscriptions` + extension `pg_net`.
- `supabase/functions/telegram-notify/index.ts`
- `supabase/functions/telegram-settings/index.ts`
- `src/pages/admin/TelegramSettings.tsx` (новая страница)
- Пункт в админ-сайдбаре + роут.
- Секрет `TELEGRAM_BOT_TOKEN` (попрошу добавить при реализации).

## Что НЕ входит в этот этап

- Двусторонний бот (команды, ответы из Telegram) — только исходящие уведомления.
- Множественные чаты/каналы — пока один глобальный chat_id (можно расширить позже до массива).
- Шаблоны сообщений, редактируемые из UI — пока зашиты в коде (тоже легко вынести позже).
