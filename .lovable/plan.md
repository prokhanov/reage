

# Промпты без доступа из AI Settings

## Текущее состояние

В базе данных (`ai_prompt_settings`) хранится **21 промпт**. Из них в UI на странице AI Settings отображаются:

- 10 промптов категорий биомаркеров (`category_*_system`, `category_*_user`) -- динамически
- 2 промпта назначений (`prescriptions_system`, `prescriptions_user`)
- 2 промпта резюме (`summary_system`, `summary_user`)
- 3 промпта стратегии (`risk_zones_*`) -- только что добавлены

**Не отображаются в UI (захардкожены как fallback или вообще не видны):**

| Ключ в базе | Назначение | Где используется |
|---|---|---|
| `health_assistant` | Промпт AI-ассистента (чат) | `health-assistant/index.ts` -- есть fallback на хардкод |
| `biological_age_system` | Системный промпт расчёта биовозраста | `analyze-biomarkers/index.ts` -- fallback "Ты эксперт по биомаркерам старения" |
| `biological_age_user` | Пользовательский промпт расчёта биовозраста | `analyze-biomarkers/index.ts` -- есть fallback |
| `trends_summary_prompt` | Промпт для описания трендов | используется где-то для генерации описания трендов |

Итого **4 промпта-сироты** -- они есть в базе, читаются из неё, но их нельзя найти и отредактировать через интерфейс AI Settings.

## Решение

Добавить в `AISettings.tsx` две новые секции:

### Секция "AI Ассистент"
- `health_assistant` -- один промпт, без пары system/user

### Секция "Биологический возраст"
- `biological_age_system` -- системный промпт
- `biological_age_user` -- пользовательский промпт

### Секция "Тренды"
- `trends_summary_prompt` -- один промпт

## Что изменится

- Все 21 промпт станут доступны для редактирования через UI
- Никаких изменений в базе данных или backend-функциях

- `src/pages/admin/AISettings.tsx` -- добавить 3 секции (~80 строк)
