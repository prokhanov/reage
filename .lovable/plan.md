## Вкладка «Рассылки» в Email-настройках

Добавляем третью вкладку рядом с «Настройки» и «Логи» — полноценный конструктор email-кампаний после регистрации пациента (drip-серии). Каждое письмо персональное, привязано к конкретному пользователю и его событию — это разрешённый транзакционный сценарий, работает на текущей инфраструктуре Lovable Emails без сторонних сервисов.

### Возможности

**Серии (drip-кампании)**
- Несколько независимых серий (например: «Онбординг после регистрации», «Напоминание сдать анализы», «Реактивация спящих»).
- Каждая серия: название, триггер запуска (регистрация / оплата / N дней без активности), глобальный вкл/выкл.
- Список шагов внутри серии: порядок, задержка от предыдущего шага (минуты/часы/дни), время отправки в часовом поясе пациента (например, «через 1 день в 10:00»), условия отмены.

**Шаг серии = одно письмо**
- Тема, прехедер (preview text).
- Тело: редактор Markdown/HTML с плейсхолдерами `{{first_name}}`, `{{dashboard_url}}`, `{{unsubscribe_url}}` и кнопкой-CTA.
- Превью прямо в админке — рендер в фирменном шаблоне ReAge (тёмный/светлый), мобайл/десктоп.
- Тестовая отправка на любой email одним кликом.
- Условия отмены шага: «не слать если оплачено», «не слать если есть анализ», «не слать если не подтвердил email», custom-выражение по полям профиля.

**База подписчиков**
- Таблица всех пациентов с колонками: имя, email, дата регистрации, статус (активен/отписан/жёстко отскочил/удалён), активные серии, последнее письмо.
- Фильтр и поиск; экспорт CSV.
- Ручные действия: запустить серию для конкретного пациента, остановить серию, сбросить прогресс.

**Отписки**
- Отдельная страница «Отписавшиеся»: email, дата, причина (одна из серий / все рассылки / жалоба / bounce).
- Гранулярная отписка: по серии и «от всех маркетинговых писем». Системные письма (auth, оплата, готов отчёт) — продолжают идти всегда.
- В подвале каждого письма серии — ссылки «Отписаться от этой серии» и «Отписаться от всех рассылок».

**Аналитика по серии**
- Сколько в очереди, отправлено, открыто (если включим pixel), пропущено по условию, ошибки.
- Воронка по шагам: сколько дошло до каждого шага.

### Архитектура

```text
Регистрация / оплата / cron-сканер «спящих»
              │
              ▼
   trigger_enroll_in_series(user, series)
              │  создаёт строки в email_drip_schedule
              ▼
     email_drip_schedule                ◄── ручное добавление из админки
     (user_id, series_id, step_id,
      send_at, status, attempt)
              │
              ▼ pg_cron каждые 5 минут
   edge fn: process-drip-emails
   ├─ проверяет email_unsubscribes (серия / все)
   ├─ проверяет cancel_conditions шага
   ├─ рендерит шаблон с плейсхолдерами
   └─ вызывает send-transactional-email
              │
              ▼
        Lovable Emails → пациент
```

### Структура базы

Новые таблицы (все RLS только для superadmin, full access для service_role):

- **`email_drip_series`** — серии: `name`, `description`, `trigger_type` (`registration` / `subscription_paid` / `inactivity`), `trigger_config` (jsonb), `is_active`.
- **`email_drip_steps`** — шаги серии: `series_id`, `order_index`, `subject`, `preheader`, `body_markdown`, `cta_label`, `cta_url`, `delay_value`, `delay_unit` (`minutes`/`hours`/`days`), `send_time_local` (например `10:00` или null = сразу), `cancel_conditions` (jsonb), `is_active`.
- **`email_drip_schedule`** — план отправок: `user_id`, `series_id`, `step_id`, `send_at`, `status` (`pending`/`sent`/`skipped`/`failed`/`cancelled`), `skip_reason`, `attempt`, `sent_at`. Уникальный индекс `(user_id, step_id)` — защита от дублей.
- **`email_unsubscribes`** — отписки: `email`, `user_id`, `scope` (`series:<id>` / `all_marketing`), `reason`, `unsubscribed_at`. Проверяется перед каждой отправкой.

Триггеры:
- `on profiles INSERT` → `enroll_in_active_series('registration')`.
- `on subscriptions status=active` → `enroll_in_active_series('subscription_paid')`.

### Edge-функции

- **`process-drip-emails`** (новая) — pg_cron каждые 5 мин: забирает `pending where send_at <= now()`, фильтрует по отпискам/условиям, рендерит, шлёт через `send-transactional-email`, обновляет статус.
- **`drip-test-send`** (новая) — отправка тестового письма на введённый email, рендерит шаг с фейковыми данными.
- **`drip-unsubscribe`** (новая) — обработчик публичной ссылки отписки по подписанному токену, возвращает страницу «вы отписаны».
- **`send-transactional-email`** — переиспользуем существующий.

Один универсальный React Email шаблон `drip-generic.tsx`, который принимает `{subject, preheader, body_html, cta_label, cta_url, unsubscribe_url}` — тело шага рендерится из Markdown в HTML на стороне функции, чтобы редактирование шло в админке без кода.

### UI: вкладка «Рассылки»

Подвкладки внутри:
1. **Серии** — карточки серий, кнопка «Создать серию», в каждой: список шагов с drag-and-drop порядком, кнопка «Добавить шаг», переключатель active.
2. **Редактор шага** (диалог/боковая панель): тема, прехедер, Markdown-редактор тела, CTA, задержка, время дня, условия отмены, кнопка «Превью» и «Отправить тест себе».
3. **Подписчики** — таблица пациентов, прогресс по сериям, поиск, экспорт CSV, ручной enrol/cancel.
4. **Отписавшиеся** — таблица, фильтр по scope, экспорт.
5. **Аналитика** — счётчики и воронка по шагам.

### Шаги реализации

1. Миграция: 4 новые таблицы + RLS + триггеры на `profiles` и `subscriptions` + индексы.
2. React Email шаблон `drip-generic.tsx` + регистрация в `registry.ts` + Markdown→HTML рендер (`marked` уже есть).
3. Edge-функции `process-drip-emails`, `drip-test-send`, `drip-unsubscribe`; деплой.
4. pg_cron job каждые 5 минут (insert tool, не миграция — содержит project ref).
5. Сидинг одной дефолтной серии «Онбординг после регистрации» с 4 шагами (приветствие, как сдать анализы +1д, зачем биовозраст +3д, напоминание +7д) — все шаги в состоянии `is_active=false`, серия `is_active=false`, чтобы ничего не ушло до ревью.
6. Frontend: новая третья вкладка в `EmailSettings.tsx` + компоненты `DripSeriesList`, `DripStepEditor`, `DripSubscribers`, `DripUnsubscribes`, `DripAnalytics`.
7. Публичная страница `/unsubscribe/:token` — обрабатывает токен, показывает «вы отписаны / выбрать другие рассылки».
8. Сценарии тестирования: тест-отправка из админки, ручной enrol тестового пациента, проверка отписки, проверка cancel-условия.

### Безопасность и анти-спам

- Все таблицы и edge-функции — только superadmin (RLS + проверка роли в функциях).
- Ссылка отписки — подписанный HMAC-токен, не угадывается.
- Жёсткий bounce/жалоба → автозапись в `suppressed_emails` (уже работает) + в `email_unsubscribes` scope=`all_marketing`.
- Лимит: не больше 1 письма серии в сутки одному пациенту (защита от случайного спама при перенастройке задержек).
- Системные письма (регистрация, оплата, готовность отчёта) **никогда не блокируются** маркетинговыми отписками.

### Что важно знать заранее

- Дефолтная серия после миграции выключена — включаете вручную после ревью текстов.
- Существующие пациенты в серию не попадут автоматически (только новые регистрации). Если нужно догнать — отдельная кнопка «Запустить для всех существующих пациентов» в админке серии.
- Открытия/клики (трекинг-пиксель, UTM в ссылках) — могу добавить, но это отдельный шаг; в первой итерации достаточно delivered/failed.
- Часовой пояс отправки берётся из `profiles.timezone` (если есть) или UTC; «отправить в 10:00» означает 10:00 по локали пациента.
