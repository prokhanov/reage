Проблема: карта `LabLocationsMap` (Leaflet) захватывает события колёсика мыши сразу при наведении, поэтому при быстром скролле страницы курсор, случайно попавший на карту, начинает зумить карту вместо прокрутки страницы.

Предлагаемое решение: зум колёсиком включается только после того, как курсор задержался над картой 500 мс. Пока пользователь просто «проскакивает» мимо карты, событие скролла продолжает идти странице.

```text
┌─────────────────────────────────────┐
│  LabLocationsMap                    │
│  scrollWheelZoom={false}            │
│                                     │
│  + DelayedScrollWheelZoom           │
│    - mouseenter → timer 500 ms      │
│    - timer done → enable zoom       │
│    - mouseleave → disable + reset   │
└─────────────────────────────────────┘
```

## Что будет сделано

1. **Новый компонент внутри `LabLocationsMap.tsx`** — `DelayedScrollWheelZoom`.
   - Использует `useMap()` из `react-leaflet`.
   - Вешает слушатели `mouseenter` / `mouseleave` на DOM-контейнер карты (`map.getContainer()`).
   - При `mouseenter` запускает `setTimeout` на 500 мс; после истечения вызывает `map.scrollWheelZoom.enable()`.
   - При `mouseleave` очищает таймер и вызывает `map.scrollWheelZoom.disable()`.
   - При размонтировании чистит таймер и слушатели.

2. **Проп `scrollWheelZoomDelay?: number`** в `LabLocationsMap`.
   - Значение по умолчанию: `500` (мс).
   - `0` — отключает задержку, зум работает сразу (как сейчас).

3. **Изменение `MapContainer`**:
   - Убрать `scrollWheelZoom` (или передать `false`), чтобы Leaflet не включал зум сразу.
   - Вместо этого внутри карты добавить `<DelayedScrollWheelZoom delay={scrollWheelZoomDelay} />`.

4. **Обновление точек использования**:
   - `src/components/landing/WhereToTestSection.tsx` — оставить задержку по умолчанию 500 мс (карта внутри скроллящегося лендинга).
   - `src/components/CallbackRequestDialog.tsx` — оставить задержку по умолчанию 500 мс (карта внутри диалога/страницы).
   - `src/pages/admin/LabLocations.tsx` и `src/components/admin/LabMapContextsTable.tsx` — передать `scrollWheelZoomDelay={0}` или оставить текущее поведение, так как это полноэкранные/админские карты, где немедленный зум ожидаем.

5. **Проверка**:
   - Быстро скроллить лендинг/диалог через карту: страница должна продолжать скроллиться, пока курсор не задержится на карте > 500 мс.
   - После 500 мс задержки колёсиком карта должна зумиться как обычно.
   - При уходе курсора с карты зум снова отключается.

## Технические детали

- Задержка через `setTimeout`, не `debounce` от событий `wheel`. Это соответствует запросу пользователя: зум активируется только после полсекундного наведения, а не после первого колеса.
- Используем `map.scrollWheelZoom.enable()` / `disable()` — стандартные методы Leaflet `ScrollWheelZoom` handler.
- События `mouseenter`/`mouseleave` не будут блокировать нативный скролл страницы; Leaflet без включённого `scrollWheelZoom` не вызывает `preventDefault` на `wheel`.
- Добавим `aria-label` / визуальный hint не требуется, но при желании можно позже добавить overlay «Зажмите для зума» — отдельно от этой задачи.
