## Что есть сейчас

Сейчас в `analyze-biomarkers` вшито жёстко:
- **Модель:** `google/gemini-2.5-flash` во всех 6 точках (категории, prescriptions, summary, snapshot, bio-age × 2)
- **Reasoning:** не используется
- **Temperature:** 0 только в snapshot и bio-age, в остальных — дефолт
- **Параметра «глубины» в UI нет** — нельзя ни выбрать модель, ни включить более тщательный режим

То есть «глубину» крутить можно — Lovable AI Gateway это поддерживает, — но никакого переключателя пока не существует.

## Что предлагаю сделать

Добавить режим **«Глубокий анализ»** (Deep Mode) — пользователь/админ выбирает его при запуске генерации, отчёт делается дольше, но качественнее.

### 1. Что меняется в edge-функции `analyze-biomarkers`

Принимаем новый необязательный параметр `mode: "standard" | "deep"` (по умолчанию `standard` — обратная совместимость).

Создаём профиль модели:

```text
standard (как сейчас):
  model:     google/gemini-2.5-flash
  reasoning: —
  retries:   2

deep (новое):
  model:     google/gemini-2.5-pro          // более сильная модель
  reasoning: { effort: "high" }              // глубокое размышление
  retries:   3                               // больше попыток на короткие ответы
  max_completion_tokens: ×1.5                // запас под reasoning-токены
```

Профиль применяем во всех 6 точках вызова (категории, prescriptions, summary, snapshot, bio-age refinement). Snapshot и bio-age сохраняют `temperature: 0` — там это правильно.

В `meta` снапшота записываем `mode` и реальную модель — будет видно в админке, чем именно сгенерирован отчёт.

### 2. Что меняется в UI

**Админ — `CreateAnalysisWizard` / `EditAnalysisWizard`:**
- Рядом с чекбоксом «Сгенерировать персональный отчет» добавляем выпадающий выбор:
  - **Стандартный** (~3–5 мин) — текущее поведение
  - **Глубокий анализ** (~8–15 мин, выше точность) — новый режим
- Выбор передаётся параметром `mode` в edge-функцию.

**Опционально (отдельно от этого плана):** дать пациенту возможность перегенерировать свой отчёт в Deep-режиме — но это уже следующий шаг, сейчас не делаем.

### 3. Что НЕ меняется

- Архитектура промптов и таблиц — нетронута.
- Логика рекомендаций (v1) — нетронута.
- Поведение по умолчанию — то же, что сейчас (`standard`).

## Ожидаемый эффект

| Параметр | Standard | Deep |
|---|---|---|
| Модель | gemini-2.5-flash | gemini-2.5-pro |
| Reasoning | нет | high |
| Время генерации | 3–5 мин | 8–15 мин |
| Стоимость (кредиты Lovable AI) | базовая | ~5–10× выше |
| Качество интерпретаций | хорошее | заметно глубже, меньше шаблонности, лучше связность между категориями |

## Файлы, которые будут затронуты

- `supabase/functions/analyze-biomarkers/index.ts` — параметр `mode`, профиль модели в 6 fetch-вызовах
- `src/components/admin/CreateAnalysisWizard.tsx` — селектор режима
- `src/components/admin/EditAnalysisWizard.tsx` — то же (если оттуда тоже запускается генерация)
- Места вызова `supabase.functions.invoke('analyze-biomarkers', ...)` — пробрасываем `mode`
