
## Проблема

Раздел «Стратегия здоровья» (`/health-strategy`) сейчас показывает данные слишком технично:
- Проценты риска (например, «47%») без объяснения, что это значит
- Стрелки ↑↓→ без подписи
- Технические единицы вроде «↑2-3 фл, ↓20-30%» — непонятно неспециалисту
- Сроки в формате «7-14 дней» без контекста «когда ждать результат»
- Сухие формулировки причин и эффектов

Нужно сделать интерфейс понятным человеку без медицинского образования: объяснить что значит каждое число, зачем мы это рекомендуем и когда ждать эффект.

## План доработок

### 1. RiskMap (`src/components/risk-zones/RiskMap.tsx`) — карточки систем организма

- Добавить подпись над процентом: вместо «Риск 47%» → **«Уровень риска: 47% — средний»** с цветовой меткой (низкий / средний / высокий)
- Рядом со стрелкой добавить текстовую расшифровку: ↑ → «Ухудшается», → → «Стабильно», ↓ → «Улучшается»
- Добавить tooltip/подсказку «?» рядом с заголовком «Уровень риска» с пояснением: «Вероятность развития проблем в этой системе на основе ваших биомаркеров. Чем ниже — тем лучше.»

### 2. SmartPriorities (`src/components/risk-zones/SmartPriorities.tsx`) — главные изменения

Переработать структуру каждой задачи (TaskCard) так, чтобы было видно 4 чётких блока с подписями:

```text
┌─────────────────────────────────────────────┐

│ Принимать витамин B12 1000 мкг утром       │
├─────────────────────────────────────────────┤

│ Через 2 недели                              │
└─────────────────────────────────────────────┘
```

Конкретно:
- Заменить иконку `ArrowRight` + сухой текст на блок с заголовком **«Что улучшится»**
- Превратить `prediction.improvement` (например «↑2-3 фл») в человеческий текст. Добавить функцию-«переводчик» технических улучшений: если в строке есть «фл», «г/л», «%» с числами — оборачивать в скобки как пояснение, а основной текст брать из `prediction.effect` («Улучшение параметров красной крови, повышение уровня энергии»). Бейдж с цифрами — мелким шрифтом как «техническая деталь».
- `task.reason` подписать заголовком **«Зачем это нужно»**
- Сроки (`timeline` / `timeline_days`) выводить в виде **«⏱ Результат через ~2 недели»** вместо просто «7-14 дней»

### 3. Блок «Ожидаемые улучшения» (внутри focus каждого уровня)

Сейчас:
```
Гемоглобин    [+5 г/л]  ⏱ 2 нед.
```

Станет:
```
Гемоглобин (переносит кислород по телу)
Сейчас → Цель: 130 → 135 г/л
Ждать результат: ~2 недели
```

- Добавить короткие народные пояснения для самых частых метрик через словарь в компоненте (Гемоглобин → «переносит кислород», Ферритин → «запас железа», ЛПНП → «плохой холестерин» и т.д.). Если метрика не в словаре — просто название без пояснения.
- Раскрыть формат: «X → Y» вместо «+N» для понятности направления

### 4. Заголовки уровней (immediate / medium_term / long_term)

Добавить под заголовком краткое объяснение **что это за уровень**:
- Краткосрочно: «Что сделать прямо сейчас, чтобы быстро почувствовать эффект»
- Среднесрочно: «Изменения для устойчивого улучшения через 1–2 месяца»
- Долгосрочно: «Стратегия для замедления старения и профилактики на годы»

### 5. Заголовок страницы (`HealthStrategy.tsx`)

Добавить под подзаголовок короткий пояснительный блок (info-карточка):
