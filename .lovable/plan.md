# Блок 2 (Как устроено сопровождение) — sticky-right timeline

Переделать `src/components/landing/v2/HowItWorksBlock.tsx` в формат «scrollytelling», как на видео: слева длинная скроллящаяся колонка текста с 4 шагами, справа — зафиксированный (sticky) визуал-таймлайн, у которого подсвечивается активный шаг в зависимости от того, какой пункт сейчас в зоне видимости.

Работает только на десктопе/планшете. На мобиле остаётся текущая вертикальная компоновка (sticky отключается, таймлайн скрыт или встроен).

## Раскладка (lg+)

```
container max-w-6xl
┌──────────────────────────┬───────────────────────┐
│ LEFT: scroll column      │ RIGHT: sticky panel   │
│ (col-span-7)             │ (col-span-5, sticky)  │
│                          │                       │
│  Заголовок секции        │   ┌─ TIMELINE ─┐      │
│                          │   │ ● Шаг 1    │      │
│  ── Шаг 1 ──────────     │   │ │          │      │
│  Заголовок + описание    │   │ ○ Шаг 2    │      │
│  (min-h ~70vh)           │   │ │          │      │
│                          │   │ ○ Шаг 3    │      │
│  ── Шаг 2 ──────────     │   │ │          │      │
│  …                       │   │ ○ Шаг 4    │      │
│                          │   └────────────┘      │
│  ── Шаг 3 ──────────     │                       │
│  …                       │ (top-24, h-screen-ish)│
│                          │                       │
│  ── Шаг 4 ──────────     │                       │
└──────────────────────────┴───────────────────────┘
```

- Левая колонка: 4 «секции-шага», каждая `min-h-[70vh]` (или `80vh`), с иконкой/номером, заголовком и описанием. Они задают скролл-длину блока.
- Правая колонка: `sticky top-24`, высота примерно `h-[80vh]`, внутри — вертикальный таймлайн со всеми 4 шагами; активный шаг выделен (зелёный кружок + усиленный фон-«пилюля», как на видео), остальные — приглушённые. Между точками — пунктирная линия.
- Активный шаг определяется через `IntersectionObserver` по левым секциям (rootMargin примерно `-40% 0px -50% 0px`, чтобы триггер был по центру вьюпорта). Состояние `activeIndex` хранится в локальном `useState`.

## Что меняется в коде

Файл `src/components/landing/v2/HowItWorksBlock.tsx`:

1. Сохранить текущий массив `items` (4 шага) и существующий заголовок секции — он становится «шапкой» левой колонки.
2. Заменить grid 4-в-ряд на двухколоночный layout: `grid lg:grid-cols-12 gap-10`, левая `lg:col-span-7`, правая `lg:col-span-5`.
3. Левая колонка: рендерим 4 `<article>` (по шагу), каждой вешаем `ref` и `data-step={i}`. Используется существующий стиль карточек (или упрощённая версия без hover-shadow, т.к. они теперь крупные «полноэкранные» секции).
4. Правая колонка (только `hidden lg:block`): `sticky top-24 self-start`. Внутри — компонент `TimelineRail` с массивом тех же `items`, принимающий `activeIndex`. Активный пункт: зелёный кружок (`bg-primary`), пилюля-подложка `bg-primary/15`, текст яркий. Неактивный — `bg-muted`, текст приглушённый. Переходы через `transition-all duration-300`.
5. Хук `useActiveStep(refs)` на `IntersectionObserver`: при пересечении секцией центра вьюпорта выставляет её индекс активным. Cleanup на unmount.
6. На мобиле (`< lg`): правая колонка скрыта, шаги отображаются как сейчас (компактные карточки одна под другой, без принудительных `min-h`).

## Прочие замечания

- Никаких новых зависимостей. Только React + Tailwind + `IntersectionObserver`.
- Цвета — только семантические токены (`primary`, `muted-foreground`, `border`, `card`), без хардкода.
- Бейдж «Блок 2» (из `LandingV2.tsx`) не трогаем.
- Анимация подсветки активного шага плавная, но без скролл-джекинга — обычный нативный скролл страницы.
