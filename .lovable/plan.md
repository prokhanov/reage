
## Цель
Убрать артефакты markdown-экранирования в финальном PDF (вида `1\.` / «1\» и маркеры `*`), чтобы в PDF отображались нормальные нумерованные и маркированные списки.

## Что происходит сейчас (почему “по прежнему есть 1\ и *”)
В `src/pages/Recommendations.tsx` функция `parseMarkdownToPdfMake`:
1) Делает `trimmedLine = line.trim()` только для проверки пустой строки,  
   но **все остальные проверки (заголовки/списки) делает по исходной `line`**.
   - Если строка списка имеет **отступы** (например `  * пункт`), regex `^[-*]\s` не срабатывает → строка уходит в “обычный текст” → в PDF остаётся `*`.
2) Нумерованные элементы часто приходят как `1\.` (markdown-escape), а наш regex ищет только `1.` (`^\d+\.\s`) → не срабатывает → в PDF остаётся `1\.`.

## Решение (концепция)
Сделать парсер “устойчивым”:
- Матчить заголовки/списки по **обрезанной строке** (`trimmedLine`), чтобы отступы не ломали распознавание.
- Для нумерованных списков поддержать вариант с экранированной точкой: `1\.`.
- При формировании текста для pdfmake убрать `\.` → `.` (как минимум в тех местах, где это встречается чаще всего: заголовки/параграфы/списки).

## План изменений в коде (1 файл)
### 1) Файл
- `src/pages/Recommendations.tsx`

### 2) Место
- Функция `parseMarkdownToPdfMake`

### 3) Правки логики построчного разбора
Внутри цикла `for (let line of lines)`:
1. Использовать `trimmedLine` для:
   - `startsWith('### ')`, `startsWith('## ')`, `startsWith('# ')`
   - проверки маркированных списков
   - проверки нумерованных списков  
   (то есть **не использовать `line` для матчинга**)

2. Маркированные списки:
   - Было: `line.match(/^[-*]\s/)`
   - Станет: `trimmedLine.match(/^[-*]\s+/)`  
   Это сразу решит проблему с `  * пункт` (отступы).

3. Нумерованные списки:
   - Было: `^\d+\.\s`
   - Станет: поддержка `\.` через regex вида:
     - проверка: `/^\d+\\?\.\s+/`
     - разбор: `/^(\d+)\\?\.\s+(.*)$/`
   Это уберёт `1\.` и преобразует в нормальный “1. …”.

4. Удаление артефакта `\.` в тексте:
   - Перед `parseInlineMarkdown(...)` для headerText / listText / paragraphText прогнать простую очистку:
     - заменить `\.` на `.`
   Важно: не трогать все `\*` глобально (чтобы случайно не сломать задуманные экранирования звёздочек в тексте и не включить курсив через наш упрощённый `parseInlineMarkdown`).

### 4) Что получится в результате
- `1\. Общий холестерин...` → `1. Общий холестерин...` (без обратного слэша)
- `  * Приоритет: ...` → станет нормальным буллетом `• Приоритет: ...` (без `*`)

## Тестирование (обязательно)
1) Открыть любой отчёт, где в тексте есть:
   - строки вида `1\.` и/или `1.`
   - строки со звёздочками `*` с отступами (как на скриншоте)
2) Экспортировать в PDF
3) Проверить, что:
   - нет `1\` / `1\.` в PDF
   - нет “сырого” `*` в начале строк (должны стать буллеты)
4) Быстрый регресс-тест: убедиться, что обычные заголовки `# / ## / ###` и текущие буллеты `-` продолжают работать.

## Возможные улучшения (необязательно, если всплывёт)
