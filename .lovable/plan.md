## Раздел «Где сдать анализы» на главной странице

### Куда вставляем

В `src/pages/Index.tsx` — новый блок `WhereToTestSection` сразу **перед `<FAQSection />`** (т. е. после `PreventiveMedicineSection`, до «Часто задаваемых вопросов»).

### Что показываем

1. **Шапка раздела** — в том же стиле, что и другие секции (`HowItWorksSection` как эталон): pill-чип «Удобно для вас», крупный H2 с градиентом, подзаголовок-описание.
2. **Два режима забора крови** — две карточки рядом, в стилистике других карточек лендинга (`rounded-3xl bg-card/50 backdrop-blur-sm border`, иконка в градиентном квадрате):
   - «На дому» — выезд медсестры, 10–15 минут, по всей Москве и МО, СПб.
   - «В клинике» — десятки партнёрских точек, удобный график, без очередей.
3. **Селектор города** над картой — сегментированный переключатель (ToggleGroup): «Москва», «Московская область», «Санкт-Петербург». Активный город:
   - фильтрует точки на карте по полю `city` (для МО — все города кроме Москвы и СПб);
   - триггерит автозум на `fitBounds` по отфильтрованным точкам (с плавной анимацией Leaflet).
   - Рядом — счётчик «N точек».
4. **Карта** — `<LabLocationsMap>`, настройки тянем из строки `lab_map_contexts.key = 'landing'` (стиль тайлов, фильтры, высота, кнопки и их подписи, дефолтный город/зум). Если контекст выключен (`is_enabled=false`) — карту не показываем, только описательную часть. Кнопка «Выбрать эту лабораторию» на лендинге не нужна — насильно скрываем (даже если в админке включена), показываем только «Перейти на сайт партнёра».

### Доступ к данным с публичной главной

Сейчас `lab_locations` и `lab_map_contexts` доступны только `authenticated`. Лендинг видят анонимы — нужна миграция:

- `lab_map_contexts`: `GRANT SELECT ... TO anon` + публичная политика чтения только включённых контекстов (`is_enabled = true`).
- `lab_locations`: `GRANT SELECT ... TO anon` + публичная политика чтения только активных точек (`is_active = true`).

Существующие политики для `authenticated` остаются без изменений.

### Логика секции

- При маунте: параллельно грузим контекст `landing` и активные точки.
- Если контекст найден и `is_enabled = true` — рендерим карту с его настройками; иначе блок без карты.
- Начальный город — `default_city` из контекста (моsсква по умолчанию).
- Переключение города → `setCity` → перефильтрованные `items` идут в карту, плюс `key={city}` форсит ремаунт `FitBounds`, чтобы карта плавно села по новым границам.
- Высота карты адаптивная: `min(context.height_px, 60vh)` на десктопе, фикс 380px на мобильном.

### Стилизация (как в остальных разделах)

- `section` с `py-12 md:py-16`, фоновые `blur` пятна `bg-primary/5` / `bg-accent/5` как в `HowItWorksSection`.
- Контейнер `container mx-auto px-4`, заголовок `text-3xl sm:text-4xl md:text-5xl font-bold` с градиентом на ключевом слове («там, где **удобно**»).
- Карточки режимов — анимация `animate-fade-in` с шагом задержки, hover-подсветка как у шагов «Как это работает».
- Селектор города и счётчик точек — выровнены по строке, на мобильном переносятся.
- Карта обёрнута в `rounded-3xl border border-border/50 overflow-hidden shadow-xl shadow-primary/5`.
