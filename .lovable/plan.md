

## Проблема

Валидация диапазонов запускается только при нажатии «Сохранить». Пока пользователь вводит данные, UI молча показывает «Общий: 55 — 5» в Оптимальной зоне без каких-либо предупреждений. Нужна валидация в реальном времени.

## Решение

### 1. Real-time валидация при изменении полей

Добавить функцию `getRangeErrors(editingBiomarker)`, которая вызывается при каждом изменении input-полей через `onChange`. Результат — массив ошибок, отображаемых под блоком Оптимальной зоны и рядом с проблемными полями.

### 2. Логика проверки

Для каждой группы (Общий, Мужчины, Женщины) проверять цепочку **только из заполненных значений**:

```text
critical_min ≤ normal_min ≤ optimal_min ≤ optimal_max ≤ normal_max ≤ critical_max
```

Дополнительно: если `optimal_min > optimal_max` — красная рамка вокруг блока Оптимальной зоны и текст ошибки.

### 3. Перевести все input-поля на управляемое состояние

Сейчас часть полей использует `defaultValue` без `onChange`, а часть — с `onChange` обновляющим `editingBiomarker`. Для real-time валидации **все** поля диапазонов должны обновлять `editingBiomarker` через `onChange`. Это позволит:
- Вычислять ошибки из `editingBiomarker` при каждом изменении
- Показывать ошибки inline без необходимости собирать formData

### 4. Визуальная индикация ошибок

- Блок «Оптимальная зона» — красная рамка и текст ошибки если `optimal_min > optimal_max`
- Под секцией диапазонов — список всех ошибок в красном Alert
- Кнопка «Сохранить» — disabled пока есть ошибки

### 5. Изменения

**Файл: `src/pages/admin/DataManagement.tsx`**
- Добавить `useMemo` для `rangeErrors` на основе `editingBiomarker`
- Добавить `onChange` ко всем 18 полям диапазонов (6 полей × 3 группы), обновляющие `editingBiomarker`
- Показывать Alert с ошибками если `rangeErrors.length > 0`
- Добавить красную рамку блоку Оптимальной зоны при ошибке
- Disable кнопки «Сохранить» при наличии ошибок
- Сохранить on-submit валидацию как safety net
