

## Проблема: confidence factor считает покрытие от ВСЕХ маркеров в системе

Пациент сдал все маркеры по своему плану (например, 25 из 25), но система считает покрытие как 25/71 (все биомаркеры в БД) = 35%. При пороге 0.5 это даёт confidenceFactor = 0.7, и индекс тянется к 70.

Это принципиально неверно: если пациент сдал 100% маркеров своего плана, confidence должен быть 1.0.

### Решение

**Файл:** `supabase/functions/analyze-biomarkers/index.ts`

Заменить знаменатель `totalBiomarkersInSystem` (все 71 маркер в системе) на количество маркеров в плане пациента. Логика:

1. Получить подписку пациента → `plan_id`
2. Посчитать маркеры в плане через `plan_biomarkers` (count where plan_id)
3. Использовать это число как знаменатель для coverage
4. Fallback: если подписки/плана нет — использовать количество сданных маркеров (markerCount), т.е. confidence = 1.0

Конкретные изменения (строки ~1211-1222):
- Вместо запроса `biomarkers.select("id", { count: "exact" })` — запрос подписки пациента и его plan_biomarkers count
- Передать это число в `calculateHealthIndex` как `totalBiomarkersInSystem`

Также обновить комментарий на строке 1169 (`× 25` → `× 15`).

### Результат

Пациент сдал все маркеры плана → coverage = 100% → confidenceFactor = 1.0 → adjusted = raw. Индекс здоровья будет точно отражать состояние без искусственного занижения.
