## Что меняем

В админке `/admin/labs` вместо общего вида карты на вкладке «Карта» показываем таблицу мест использования карты в продукте. Каждая строка — отдельная сущность (контекст) со своими настройками. Настройки и превью открываются в модальном диалоге поверх таблицы. Реальная вставка карты на лендинг и в ЛК пациента в этой итерации **не делается** — только сохранение настроек.

## Таблица «Где используется»

Колонки:
- Название (напр. «Главная страница», «Личный кабинет пациента»)
- Расположение (текст, напр. «Лендинг → блок Лаборатории», «ЛК → раздел Анализы»)
- Город по умолчанию (Москва / Санкт-Петербург)
- Статус (Включено / Выключено)
- Действия: «Настроить» (открывает диалог)

Список контекстов на старте (предзаполняется сидом):
1. Главная страница (landing)
2. Личный кабинет пациента (patient_dashboard)

Добавления/удаления строк в UI пока нет — фиксированный набор.

## Диалог настроек контекста

Открывается поверх по клику «Настроить». Содержимое:

- Город по умолчанию — селект Москва / Санкт-Петербург (по умолчанию Москва)
- Зум по умолчанию — число, автоматически подставляется под город (Москва ~10, СПб ~11), можно переопределить
- Показывать только активные лаборатории — переключатель
- Высота карты (px) — число
- Превью карты — встроенный `LabLocationsMap` с применёнными настройками (центр/зум по выбранному городу, точки из `lab_locations`)

Кнопки: «Отмена», «Сохранить». Сохранение пишет настройки в БД, диалог закрывается, в таблице обновляются значения.

## Технические детали

**БД:** новая таблица `lab_map_contexts`
- `key` (text, unique) — `landing` | `patient_dashboard`
- `name` (text)
- `location` (text)
- `default_city` (text, `moscow`|`spb`, default `moscow`)
- `default_zoom` (int, default 10)
- `only_active` (bool, default true)
- `height_px` (int, default 420)
- `is_enabled` (bool, default false)
- стандартные id/created_at/updated_at

RLS: чтение `authenticated`, запись — только superadmin/admin с правами на модуль `lab_locations`. GRANT на `authenticated` и `service_role`.

Сид при миграции — две строки (`landing`, `patient_dashboard`).

**Фронт:**
- В `src/pages/admin/LabLocations.tsx` на вкладке «Карта» заменить текущий рендер `LabLocationsMap` на новый компонент `LabMapContextsTable`.
- Новый файл `src/components/admin/LabMapContextsTable.tsx` — таблица + dialog `LabMapContextSettingsDialog`.
- Координаты центров: Москва `[55.7558, 37.6173]`, СПб `[59.9343, 30.3351]`. Передаются в `LabLocationsMap` как новые опциональные пропсы `center` и `zoom` (текущий компонент использовать как есть, добавить эти два проп-параметра).
- Сохранение через `supabase.from('lab_map_contexts').update(...).eq('key', ...)`.
- Никаких изменений в лендинге и ЛК пациента в этой итерации.
