## План: сверка `ai_prompts_backup.json` с таблицей `ai_prompt_settings`

Идём по порядку из файла, сверху вниз. Для каждого ключа:

1. Читаю `prompt_text` из файла.
2. Читаю `prompt_text` из БД (`ai_prompt_settings` по `key`).
3. Если тексты идентичны — пишу «без изменений», иду дальше без вопросов.
4. Если есть разница — присылаю саммари:
   - `key`
   - краткое описание сути изменений (что добавлено / убрано / переформулировано)
   - при необходимости — компактный diff ключевых блоков
5. Жду твоего «ок» → выполняю `UPDATE ai_prompt_settings SET prompt_text = ... WHERE key = ...` через миграцию.
6. Если ключа нет в БД — отдельно спрашиваю, создавать ли (INSERT) или пропустить.
7. Ключи, которые есть в БД, но отсутствуют в файле, не трогаем (файл — не источник правды на удаление).

Сравниваем только `prompt_text`. `description` и прочие поля не трогаем.

После твоего «поехали» перехожу в build mode, начинаю с первого ключа в файле и присылаю первую сверку.
